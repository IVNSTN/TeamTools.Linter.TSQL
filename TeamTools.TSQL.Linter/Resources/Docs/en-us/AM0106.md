

# Ambiguous intent about NULL / NOT NULL

|||
|-|-|
| Id | **AM0106**
| Mnemo | COLUMN_NULLABLE
| Severity | ⚠ Warning
| Category | [Ambiguity](./Category_Ambiguity.md)
| Source code | [NullNotNullColumnRule.cs](../../../Rules/Ambiguity/NullNotNullColumnRule.cs)

## Cause

<p id="descr">This rule is triggered when NULL or NOT NULL is not specified for the declaration of table column.</p>

## Examples

Bad

```sql
CREATE TABLE dbo.Employee (EmployeeId INT);

-- database with ANSI_NULL_DEFAULT ON
DECLARE @Employee TABLE (EmployeeId INT);
```

Good

```sql
CREATE TABLE dbo.Employee (EmployeeId INT NOT NULL);

DECLARE @Employee TABLE (EmployeeId INT NULL);
```

## Tips

💡 Do not rely on the database option ANSI_NULL_DEFAULT, as it can be overridden by the connection-level setting ANSI_NULL_DEFAULT. You should always set the column nullability explicitly to avoid confusion.
