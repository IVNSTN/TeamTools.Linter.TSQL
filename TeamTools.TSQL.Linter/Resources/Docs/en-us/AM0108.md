

# Object creation with no schema defined

|||
|-|-|
| Id | **AM0108**
| Mnemo | CREATE_WITH_SCHEMA
| Severity | ⚠ Warning
| Category | [Ambiguity](./Category_Ambiguity.md)
| Source code | [SchemaQualifiedObjectCreationRule.cs](../../../Rules/Ambiguity/SchemaQualifiedObjectCreationRule.cs)

## Cause

This rule is triggered when the schema name is skipped in the object creation.
<p id="descr">Specify schema name.</p>

## Examples

Bad

```sql
CREATE TABLE employee ...
CREATE VIEW hiredate_view ...
```

Good

```sql
CREATE TABLE dbo.employee ...
CREATE VIEW dbo.hiredate_view ...
```

## Tips

💡 Different users may have different DEFAULT_SCHEMA, except members of the sysadmin. You should always specify the schema in the object creation scripts, because the schema should not depend on the user who will run the script.
