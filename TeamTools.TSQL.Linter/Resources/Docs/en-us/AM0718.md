

# Possible ambiguity: object type should be provided

|||
|-|-|
| Id | **AM0718**
| Mnemo | OBJECT_ID_WITHOUT_TYPE
| Severity | ⚠ Warning
| Category | [Ambiguity](./Category_Ambiguity.md)
| Source code | [ObjectIdNeedsTypeRule.cs](../../../Rules/Ambiguity/ObjectIdNeedsTypeRule.cs)

## Cause

This rule is triggered if the object type is not specified when calling the `OBJECT_ID` function.
<p id="descr">Specify the value for the object type parameter.</p>

## Examples

Bad

```sql
SELECT OBJECT_ID('dbo.my_object')
```

Good

```sql
SELECT OBJECT_ID('dbo.my_object', 'U')
```

## Tips

💡 The `OBJECT_ID` function may not find the object you were looking for if the object type is not specified. You must specify the type, for example, for tables it will be `OBJECT_ID('dbo.my_table', 'U')`.

💡 The list of object types is contained in the type column of the sys.objects view.
