

# Use DATEPART function instead of YEAR(), MONTH(), DAY() 

|||
|-|-|
| Id | **CD0277**
| Mnemo | DATE_FN_IN_CONSTRAINT
| Severity | ⚠ Warning
| Category | [ContinuousDelivery](./Category_ContinuousDelivery.md)
| Source code | [DateFunctionsInConstraintRule.cs](../../../Rules/ContinuousDeployment/DateFunctionsInConstraintRule.cs)

## Cause

It is triggered due to the use of the functions YEAR(), MONTH(), DAY() in constraints. Does not react on temp tables and table variables.
<p id="descr">Use DATEPART() with the needed parameter.</p>

## Examples

Bad

```sql
CREATE TABLE foo 
(
    bar DATETIME NOT NULL
    , mar AS YEAR(bar)
);
```

Good

```sql
CREATE TABLE foo 
(
    bar DATETIME NOT NULL
    , mar AS DATEPART(YEAR, bar)
);
```

## Tips

💡 The functions themselves are valid, but DacFx converts them to DATEPART when publishing, and so it turns out an "eternal diff".
