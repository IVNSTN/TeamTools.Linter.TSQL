

# Unexpected code block in script root

|||
|-|-|
| Id | **CD0726**
| Mnemo | CODE_IN_SCRIPT_ROOT
| Severity | ⚠ Warning
| Category | [ContinuousDelivery](./Category_ContinuousDelivery.md)
| Source code | [RootScriptBlockRule.cs](../../../Rules/ContinuousDeployment/RootScriptBlockRule.cs)

## Cause

Only object definitions can be in the code of SSDT projects
<p id="descr">Remove the extraneous code from the script.</p>

## Examples

Bad

```sql
CREATE TABLE foo 
(
    bar INT NOT NULL CONSTRAINT cns DEFAULT 0
    , mar INT NOT NULL
)
GO

INSERT INTO foo (bar, mar)
VALUES (1, 2)
```

Good

```sql
CREATE TABLE foo 
(
    bar INT NOT NULL CONSTRAINT cns DEFAULT 0
    , mar INT NOT NULL
)
GO
```

## Tips

💡 Store the pre- and post-deploy scripts in separate folders and add them to the linter exceptions. Other code must be inside stored procedures, triggers, and functions.
