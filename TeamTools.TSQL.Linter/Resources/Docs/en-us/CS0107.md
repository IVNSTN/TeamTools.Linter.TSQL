

# Named constraint in temp (#) table

|||
|-|-|
| Id | **CS0107**
| Mnemo | TMP_NAMED_CONSTRAINT
| Severity | ⚠ Warning
| Category | [CodeSmell](./Category_CodeSmell.md)
| Source code | [ConstraintNameInTempTableRule.cs](../../../Rules/CodeSmell/ConstraintNameInTempTableRule.cs)

## Cause

<p id="descr">Во временных таблицах (#) нельзя именовать констрейнты, т.к. они хранятся не в контексте таблицы, а в контексте tempdb и можно легко пересечься по названиям (будет ошибка про уникальность имени констрейнта) с другим процессом, хотя сами временные таблицы могут быть изолированы и "невидимы" между этими процессами.</p>
