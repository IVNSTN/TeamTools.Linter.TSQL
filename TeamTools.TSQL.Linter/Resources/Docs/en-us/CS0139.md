

# Trigger ordering specified

|||
|-|-|
| Id | **CS0139**
| Mnemo | TRIGGER_ORDERING
| Severity | ⚠ Warning
| Category | [CodeSmell](./Category_CodeSmell.md)
| Source code | [TriggerOrderRule.cs](../../../Rules/CodeSmell/TriggerOrderRule.cs)

## Cause

This rule is triggered if an attempt to set the trigger execution order using `sp_settriggerorder` is detected.
<p id="descr">Remove the trigger execution order setting.</p>

## Examples

Bad

```sql
EXECUTE sp_settriggerorder
    @triggername = N'[mrgn].[TR_ActPosShort_IU_repl]'
    , @order = N'last'
    , @stmttype = N'insert';
```

## Tips

💡 Specifying the order of trigger execution is not supported by DacFx.

💡 Typically, there is only one trigger on a table, and specifying the order of execution does not make any sense.
