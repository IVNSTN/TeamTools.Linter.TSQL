

# Table-level constraint is defined not at table level

|||
|-|-|
| Id | **CS0266**
| Mnemo | TABLE_LEVEL_CONSTRAINT_IN_COL
| Severity | ⚠ Warning
| Category | [CodeSmell](./Category_CodeSmell.md)
| Source code | [TableLevelConstraintInColumnRule.cs](../../../Rules/CodeSmell/TableLevelConstraintInColumnRule.cs)

## Cause

This rule is triggered when table-level constraint isn's separated by a comma from the nearest column.
<p id="descr">Separate constraint from the nearest column by a comma.</p>

## Examples

Bad

```sql
CREATE TABLE dbo.foo
(
    group_id INT NULL FOREIGN KEY (group_id) REFERENCES dbo.foo_group(id)
)
```

Good

```sql
CREATE TABLE dbo.foo
(
    group_id INT NULL,
    CONSTRAINT FK_GROUP FOREIGN KEY (group_id) REFERENCES dbo.foo_group(id)
)
```

## Tips

💡 The rule doesn't check temporary tables and table variables.

💡 The rule checks:

- PRIMARY KEY
- FOREIGN KEY
- CHECK
- UNIQUE
