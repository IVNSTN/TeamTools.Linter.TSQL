
# Look-alike char mix in string literal

|||
|-|-|
| Id | **CS0834**
| Mnemo | LITERAL_LOOK_ALIKE_CHAR
| Severity | ℹ Hint
| Category | [CodeSmell](./Category_CodeSmell.md)
| Source code | [LiteralContainsLookAlikeCharRule.cs](../../../Rules/CodeSmell/LiteralContainsLookAlikeCharRule.cs)

## Cause

This rule is triggered if a word in the string literal, typed primarily in characters of one alphabet, contains a character from another alphabet that has a visually similar counterpart in the first alphabet.
<p id="descr">Rewrite using characters of only one alphabet.</p>

## Examples

Bad

```sql
SELECT '', NULL, 'аdsf'; -- unexpected cyrillic 'а' in latin string

DECLARE @str VARCHAR(100) = 'Cтрока'; -- unexpected latin 'C' in cyrillic string

EXEC dbo.foo @arg = 'qwerty zxсv 123'; -- unexpected cyrillic 'с' in latin string
```

Good

```sql
SELECT '', NULL, 'adsf'; -- latin only

DECLARE @str VARCHAR(100) = 'АБВГД'; -- cyrillic only

EXEC dbo.foo @arg = 'qwerty zxcv 123'; -- lating + numbers

PRINT 'Переменной @c присвоено неверное значение ''A''!'; -- valid mix of c/с, A/А
```

## Tips

💡 String literals can be used not only in code branching logic or value checks, but also as constant values for insertion into a tables (from a stored procedure or a pre-deploy script). So an incorrect literal may end up in the production data and it may not be so easy to eliminate it from there.
