

# Argument does not have requested details

|||
|-|-|
| Id | **CS0905**
| Mnemo | VAR_LACKS_PRECISION
| Severity | ⚠ Warning
| Category | [CodeSmell](./Category_CodeSmell.md)
| Source code | [VariableTypePrecisionLackRule.cs](../../../Rules/CodeSmell/VariableTypePrecisionLackRule.cs)

## Cause

This rule is triggered when date or time argument doesn't have requested details (DATEPART).
<p id="descr">Remove non-existent details.</p>

## Examples

Bad

```sql
DECLARE
    @tm  TIME
    , @day DATE

SELECT
    YEAR(@tm),
    DATEPART(SECOND, @day);
```

Good

```sql
DECLARE
    @tm  TIME
    , @day DATE

SELECT
    YEAR(@day),
    DATEPART(SECOND, @tm);
```

## Tips

💡 Validate the compatibility of variable types and function arguments for DATEADD, DATEDIFF, EOMONTH and similar.

💡 Reminder:

- DATE - doesn't have time
- TIME - doesn't have date
- SMALLDATETIME - doesn't have seconds
- DATETIME - doesn't have micro- and nanoseconds
