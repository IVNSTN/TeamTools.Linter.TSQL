

# Parameter is expected to be OUTPUT

|||
|-|-|
| Id | **CS0922**
| Mnemo | PARAM_EXPECTED_AS_OUTPUT
| Severity | ⚠ Warning
| Category | [CodeSmell](./Category_CodeSmell.md)
| Source code | [BuiltInProcsArgumentOutputRule.cs](../../../Rules/CodeSmell/BuiltInProcsArgumentOutputRule.cs)

## Cause

The rule is triggered if the OUTPUT keyword is not specified for the output parameter when calling the stored procedure.
<p id="descr">Specify the OUTPUT keyword.</p>

## Examples

Bad

```sql
CREATE PROCEDURE dbo.sp_foo
      @param1  INT = NULL
    , @param2  INT = NULL OUTPUT
...

DECLARE @p2 INT;
EXEC dbo.sp_foo @param2 = @p2;
```

Good

```sql
CREATE PROCEDURE dbo.sp_foo
      @param1  INT = NULL
    , @param2  INT = NULL OUTPUT
...

DECLARE @p2 INT;
EXEC dbo.sp_foo @param2 = @p2 OUTPUT;
```

## Tips

💡 If you do not explicitly specify the OUTPUT keyword for the output parameter, the code will work, but the value of the variable will not be set.
