

# Input parameter value overwritten without attempt to read it

|||
|-|-|
| Id | **CS0924**
| Mnemo | PARAM_VALUE_IGNORED
| Severity | ℹ Hint
| Category | [CodeSmell](./Category_CodeSmell.md)
| Source code | [ParameterValueIgnoredRule.cs](../../../Rules/CodeSmell/ParameterValueIgnoredRule.cs)

## Cause

The rule is triggered if the value of an input parameter of a stored procedure or function is overwritten without reading it.
<p id="descr">Change the logic of overriding the input parameter.</p>

## Examples

Bad

```sql
CREATE PROCEDURE dbo.sp_foo
      @param1  INT
AS
    SET @param1 = 100;
...
```

Good

```sql
CREATE PROCEDURE dbo.sp_foo
      @param1  INT
AS
    SET @param1 = @param1 * 100;
...
```

## Tips

💡 It should be clear inside the procedure or function why we are changing the input parameter. If the overwriting of a parameter has no logic, then this parameter becomes useless.
