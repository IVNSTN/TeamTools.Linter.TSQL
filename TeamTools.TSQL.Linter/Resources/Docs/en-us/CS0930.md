

# Cursor name reused

|||
|-|-|
| Id | **CS0930**
| Mnemo | NAME_REUSED_CURSOR
| Severity | ⚠ Warning
| Category | [CodeSmell](./Category_CodeSmell.md), [Cursors](./Group_Cursors.md)
| Source code | [ReusedCursorNameRule.cs](../../../Rules/CodeSmell/ReusedCursorNameRule.cs)

## Cause

The rule is triggered if the cursor name in the script is reused for another cursor.
<p id="descr">Give an unique name for each cursor.</p>

## Examples

Bad

```sql
DECLARE base_cursor CURSOR
    FOR SELECT * FROM dbo.table_1
...

DECLARE base_cursor CURSOR
    FOR SELECT * FROM dbo.table_2
...
```

Good

```sql
DECLARE first_cursor CURSOR
    FOR SELECT * FROM dbo.table_1
...

DECLARE second_cursor CURSOR
    FOR SELECT * FROM dbo.table_2
...
```

## Tips

💡 Reusing a cursor name can lead to confusion and unexpected results in complex code, as it may cause the code to compile successfully but run with a different cursor than intended.
