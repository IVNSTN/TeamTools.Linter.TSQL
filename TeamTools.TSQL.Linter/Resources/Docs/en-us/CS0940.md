

# Loop predicate never changes inside loop

|||
|-|-|
| Id | **CS0940**
| Mnemo | LOOP_PREDICATE_IMMUTABLE
| Severity | ℹ Hint
| Category | [CodeSmell](./Category_CodeSmell.md)
| Source code | [LoopPredicateImmutableRule.cs](../../../Rules/CodeSmell/LoopPredicateImmutableRule.cs)

## Cause

The rule is triggered if the WHILE condition contains variables that are not changed within the loop.
<p id="descr">Check the variables from the loop condition.</p>

## Examples

Bad

```sql
DECLARE @rid INT = 1;
WHILE(@rid <= 10)
BEGIN
...
END
```

Good

```sql
DECLARE @rid INT = 1;
WHILE(@rid <= 10)
BEGIN
...
 SET @rid += 1;
END
```

## Tips

💡 The rule does not apply to predicates such as 1=1.
