

# Last statement in a function must be RETURN

|||
|-|-|
| Id | **CS0950**
| Mnemo | FUNCTION_ENDS_WITH_RETURN
| Severity | ⚠ Warning
| Category | [CodeSmell](./Category_CodeSmell.md)
| Source code | [FunctionEndsWithReturnRule.cs](../../../Rules/CodeSmell/FunctionEndsWithReturnRule.cs)

## Cause

The rule is triggered if there is no RETURN statement at the end of the function definition.
<p id="descr">Add the RETURN statement.</p>

## Examples

Bad

```sql
CREATE FUNCTION dbo.my_func (@id INT)
RETURNS INT
AS
BEGIN
...
END
```

Good

```sql
CREATE FUNCTION dbo.my_func (@id INT)
RETURNS INT
AS
BEGIN
...
RETURN (@res);
END
```

## Tips

💡 This rule is similar in meaning to rule [CS0161](./CS0161.md), which concerns unreachable code.
