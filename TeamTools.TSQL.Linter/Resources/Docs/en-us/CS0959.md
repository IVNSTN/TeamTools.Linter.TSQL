

# Duplicate expression in conditional flow has no effect

|||
|-|-|
| Id | **CS0959**
| Mnemo | DUP_CONDITIONAL_FLOW
| Severity | ⚠ Warning
| Category | [CodeSmell](./Category_CodeSmell.md)
| Source code | [DuplicateConditionRule.cs](../../../Rules/CodeSmell/DuplicateConditionRule.cs)

## Cause

The rule is triggered if the code contains mutually exclusive conditional transitions (CASE, IF-ELSE-IF) and the same conditions occur within them.
<p id="descr">Check the logic of the conditional operator or expression.</p>

## Examples

Bad

```sql
SELECT
    CASE
        WHEN @a > @b THEN 1
        WHEN @a > @b THEN 2
        WHEN @b < @a THEN 4
        ...
    END
...
```

Good

```sql
SELECT
    CASE
        WHEN @a > @b THEN 1
        ...
    END
...
```
