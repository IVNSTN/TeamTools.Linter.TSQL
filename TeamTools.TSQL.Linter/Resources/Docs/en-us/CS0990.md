

# Keyword is used for transaction name

|||
|-|-|
| Id | **CS0990**
| Mnemo | SUSPICIOUS_TRAN_NAME
| Severity | ⚠ Warning
| Category | [CodeSmell](./Category_CodeSmell.md)
| Source code | [SuspiciousTranNameRule.cs](../../../Rules/CodeSmell/SuspiciousTranNameRule.cs)

## Cause

This rule is triggered when keyword is used for transaction name.
<p id="descr">Give a transaction name that does not contain keywords.</p>

## Examples

Bad

```sql
BEGIN TRY
    BEGIN TRAN

    SELECT 1/0

    COMMIT TRAN
END TRY
BEGIN CATCH
    ROLLBACK TRAN
    THROW -- There won't throw division by zero exception,
          -- but an attemp to close the transaction with the name "THROW"
END CATCH
```

Good

```sql
BEGIN TRY
    BEGIN TRAN

    SELECT 1/0

    COMMIT TRAN
END TRY
BEGIN CATCH
    ROLLBACK TRAN;
    THROW
END CATCH
```

## Tips

💡 The rule checks:

- BEGIN TRAN
- SAVE TRAN
- COMMIT TRAN
- ROLLBACK TRAN
