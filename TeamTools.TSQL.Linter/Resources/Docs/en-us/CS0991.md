

# WAITFOR DELAY usage is not welcome

|||
|-|-|
| Id | **CS0991**
| Mnemo | WAITFOR_DELAY_USED
| Severity | ℹ Hint
| Category | [CodeSmell](./Category_CodeSmell.md)
| Source code | [WaitForDelayUsedRule.cs](../../../Rules/CodeSmell/WaitForDelayUsedRule.cs)

## Cause

This rule is triggered when WAITFOR DELAY is used.
<p id="descr">Remove WAITFOR DELAY.</p>

## Examples

Bad

```sql
CREATE PROC dbo.test
AS
BEGIN
    WAITFOR DELAY '00:01'
    RETURN 1
END
```

Good

```sql
CREATE PROC dbo.test
AS
BEGIN
    RETURN 1
END
```

## Tips

💡 WAITFOR DELAY can be useful to retry deadlock processing. In this case add stored procedure to white list.

💡 Waiting asynchronous mechanisms like Service Broker is not a database job.
