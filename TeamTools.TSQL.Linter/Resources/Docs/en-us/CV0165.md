
# Permission in object definition file

|||
|-|-|
| Id | **CV0165**
| Mnemo | PERMISSION_IN_DEFINITION
| Severity | ⚠ Warning
| Category | [CodingConvention](./Category_CodingConvention.md)
| Source code | [PermissionsInDefinitionFileRule.cs](../../../Rules/CodingConvention/PermissionsInDefinitionFileRule.cs)

## Cause

This rule is triggered if the object definition file contains instructions to set access permissions.
<p id="descr">Move permissions granting to the appropriate role file.</p>

## Examples

Bad

```sql
/* dbo.Bar.sql */
CREATE SYNONYM Bar
FOR FAR;
GO

GRANT SELECT ON OBJECT::dbo.Bar TO Foo;
GO
```

Good

```sql
/* dbo.Bar.sql */
CREATE SYNONYM Bar
FOR FAR;
GO

/* Foo.sql */
CREATE ROLE Foo ...
...

GRANT SELECT ON OBJECT::dbo.Bar TO Foo;
GO
```
