
# Non-literal output for EXISTS

|||
|-|-|
| Id | **CV0173**
| Mnemo | EXISTS_NONLITERAL_SELECT
| Severity | ⚠ Warning
| Category | [CodingConvention](./Category_CodingConvention.md)
| Source code | [ExistsHasNonLiteralSelectElementRule.cs](../../../Rules/CodingConvention/ExistsHasNonLiteralSelectElementRule.cs)

## Cause

This rule is triggered if any calculations are performed or specific fields are listed for `EXISTS (SELECT ...)` expression.
<p id="descr">For 'EXISTS', use a literal output.</p>

## Examples

Bad

```sql
IF NOT EXISTS (SELECT zzz.xxx WHERE 0 = 1)
    SELECT a
    FROM t
    WHERE EXISTS (SELECT a.id)
        OR EXISTS (SELECT tt.name AS nm FROM tt WHERE tt.name = 'asfd')
        AND NOT EXISTS (SELECT b FROM yy);
```

Good

```sql
IF NOT EXISTS (SELECT 0 WHERE 0 = 1)
    SELECT a
    FROM t
    WHERE EXISTS (SELECT 1)
        OR EXISTS (SELECT 0 FROM tt WHERE tt.name = 'asfd')
        AND NOT EXISTS (SELECT NULL FROM yy);
```
