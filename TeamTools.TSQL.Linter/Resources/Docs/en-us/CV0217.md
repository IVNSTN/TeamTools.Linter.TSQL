
# Order of column properties

|||
|-|-|
| Id | **CV0217**
| Mnemo | COLUMN_NULLABILITY_BEFORE_CONSTRAINT
| Severity | ⚠ Warning
| Category | [CodingConvention](./Category_CodingConvention.md)
| Source code | [ColumnNullableOptionBeforeConstraintRule.cs](../../../Rules/CodingConvention/ColumnNullableOptionBeforeConstraintRule.cs)

## Cause

This rule is triggered when the order of properties in a table column definition is incorrect.
<p id="descr">Specify the column properties in the correct order.</p>

## Examples

Bad

```sql
CREATE TABLE foo
(
    bar    INT         IDENTITY(1, 1) NOT NULL
    , far  VARCHAR(10) CONSTRAINT ck CHECK (CONVERT(VARCHAR(3), far) = 'zar' OR far = 'jar') NOT NULL
    , mar  DATETIME    DEFAULT CONVERT(DATE, GETDATE()) NOT NULL
);
```

Good

```sql
CREATE TABLE foo
(
    bar    INT         NOT NULL IDENTITY(1, 1)
    , far  VARCHAR(10) NOT NULL CONSTRAINT ck CHECK (CONVERT(VARCHAR(3), far) = 'zar' OR far = 'jar')
    , mar  DATETIME    NOT NULL DEFAULT CONVERT(DATE, GETDATE())
);
```

## Tips

💡 Correct order of properties in the table column definition:

1. datatype
2. `NULL`/`NOT NULL`
3. everything else (`IDENTITY`, `CONSTRAINT`, etc.)
