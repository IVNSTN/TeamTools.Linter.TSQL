
# Transaction control statements written not in two words

|||
|-|-|
| Id | **CV0224**
| Mnemo | TRAN_CONTROL_TWO_WORDS
| Severity | ⚠ Warning
| Category | [CodingConvention](./Category_CodingConvention.md)
| Source code | [TransactionControlInTwoWordsRule.cs](../../../Rules/CodingConvention/TransactionControlInTwoWordsRule.cs)

## Cause

This rule is triggered if the transaction control statement is not written in two keywords or mixes the full and abbreviated spellings of the keywords throughout the code.
<p id="descr">Write transaction control statements in two keywords, using the same spelling throughout the code.</p>

## Examples

Bad

```sql
BEGIN TRAN;
COMMIT TRANSACTION;
ROLLBACK;
```

Good

```sql
BEGIN TRAN;
COMMIT TRAN;
ROLLBACK TRAN;

-- or
BEGIN TRANSACTION;
COMMIT TRANSACTION;
ROLLBACK TRANSACTION;
```
