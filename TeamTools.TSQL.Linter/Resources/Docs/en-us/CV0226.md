
# Multiline element in DECLARE

|||
|-|-|
| Id | **CV0226**
| Mnemo | MULTILINE_DECLARE_ELEMENT
| Severity | ⚠ Warning
| Category | [CodingConvention](./Category_CodingConvention.md)
| Source code | [SingleLineDeclareElementRule.cs](../../../Rules/CodingConvention/SingleLineDeclareElementRule.cs)

## Cause

This rule is triggered when an assignment of a multi-line value is detected inside a `DECLARE` block.
<p id="descr">Simplify the assigned value or move it beyond the 'DECLARE' block.</p>

## Examples

Bad

```sql
DECLARE
    @foo INT = (1 - 2
    * 3)
    , @bar VARCHAR(10) = 'aasdf
                            asdf
                                xcvzxv'
    , @dt DATETIME = (
        SELECT next_date
        FROM calendar
        WHERE dt = DATEADD(DD, -1, GETDATE())
    );
```

Good

```sql
DECLARE 
    @foo INT = (1 - 2 * 3)
    , @bar VARCHAR(10) = ''
    , @dt DATETIME = DATEADD(DD, -1, GETDATE());
```
