

# User-defined functions in the DECLARE block

|||
|-|-|
| Id | **CV0279**
| Mnemo | UDF_IN_DECLARE
| Severity | ⚠ Warning
| Category | [CodingConvention](./Category_CodingConvention.md)
| Source code | [UserDefinedFunctionInDeclareRule.cs](../../../Rules/CodingConvention/UserDefinedFunctionInDeclareRule.cs)

## Cause

This rule is triggered when a user-defined function is called in the DECLARE block.
<p id="descr">Move the call to a SET or SELECT after the DECLARE block.</p>

## Examples

Bad

```sql
DECLARE
    @var INT = (SELECT dbo.max_var(1))
    , @dt DATETIME = dbo.f_today()
    , @int INT = my.f_last_update()
```

Good

```sql
DECLARE
    @var INT
    , @dt DATETIME
    , @int INT

SELECT
    @var = dbo.max_var(1)
    , @dt = dbo.f_today()
    , @int = my.f_last_update()
```
