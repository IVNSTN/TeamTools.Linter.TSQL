

# Integer literal '1' is expected

|||
|-|-|
| Id | **CV0294**
| Mnemo | FIXED_TRIVIAL_SCALAR
| Severity | ⚠ Warning
| Category | [CodingConvention](./Category_CodingConvention.md)
| Source code | [FixedTrivialScalarExpressionRule.cs](../../../Rules/CodingConvention/FixedTrivialScalarExpressionRule.cs)

## Cause

This rule is triggered when row counter doesn't use COUNT(1) or EXISTS doesn't use SELECT 1.
<p id="descr">Use the integer literal '1' for COUNT or EXISTS.</p>

## Examples

Bad

```sql
SELECT COUNT(*) FROM foo;
SELECT 1 WHERE EXISTS(SELECT a FROM dbo.bar);
```

Good

```sql
SELECT COUNT(1) FROM foo;
SELECT 1 WHERE EXISTS(SELECT 1 FROM dbo.bar);
```
