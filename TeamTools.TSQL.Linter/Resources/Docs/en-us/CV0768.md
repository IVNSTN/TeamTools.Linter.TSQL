

# Index column sort order isn't explicitly specified

|||
|-|-|
| Id | **CV0768**
| Mnemo | INMEMORY_INDEX_SORTING
| Severity | ⚠ Warning
| Category | [CodingConvention](./Category_CodingConvention.md), [Indices](./Group_Indices.md), [InMemory](./Group_InMemory.md)
| Source code | [InMemoryIndexSortingRule.cs](../../../Rules/CodingConvention/InMemoryIndexSortingRule.cs)

## Cause

This rule is triggered when sort order isn't explicitly specified for in-memory table index column.
<p id="descr">Specify a sort order for index column.</p>

## Examples

Bad

```sql
CREATE TABLE dbo.foo
(
    major_id        BIGINT          NOT NULL
    , secondary_id  INT             NOT NULL
    , INDEX ix_secondary_id NONCLUSTERED (secondary_id DESC, major_id) WITH (BUCKET_COUNT = 5000000)
)
WITH (MEMORY_OPTIMIZED = ON);
```

Good

```sql
CREATE TABLE dbo.foo
(
    major_id        BIGINT          NOT NULL
    , secondary_id  INT             NOT NULL
    , INDEX ix_secondary_id NONCLUSTERED (secondary_id DESC, major_id ASC) WITH (BUCKET_COUNT = 5000000)
)
WITH (MEMORY_OPTIMIZED = ON);
```
