

# PRINT in business-logic

|||
|-|-|
| Id | **CV0805**
| Mnemo | PRINTING
| Severity | ℹ Hint
| Category | [CodingConvention](./Category_CodingConvention.md)
| Source code | [PrintRule.cs](../../../Rules/CodingConvention/PrintRule.cs)

## Cause

This rule is triggered when PRINT is used in business-logic.
<p id="descr">Remove PRINT.</p>

## Examples

Bad

```sql
DECLARE @i INT = 1;
SELECT @i = @i + 5;

PRINT 'i = ' + CAST(@i AS VARCHAR(11));
```

Good

```sql
DECLARE @i INT = 1;
SELECT @i = @i + 5;
```

## Tips

💡 PRINT is acceptable in tests.
