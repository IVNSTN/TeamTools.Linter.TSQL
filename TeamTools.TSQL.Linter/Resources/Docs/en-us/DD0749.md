

# Variable-length column with small size

|||
|-|-|
| Id | **DD0749**
| Mnemo | SHORT_VAR_LENGTH_COL
| Severity | ℹ Hint
| Category | [DatabaseDesign](./Category_DatabaseDesign.md)
| Source code | [ShortVariableLengthColumnRule.cs](../../../Rules/DatabaseDesign/ShortVariableLengthColumnRule.cs)

## Cause

This rule is triggered when variable-length column with small size is used.
<p id="descr">Use fixed-length column.</p>

## Examples

Bad

```sql
CREATE TABLE dbo.foo
(
    id INT NOT NULL
    , code VARCHAR(2) NOT NULL
);
```

Good

```sql
CREATE TABLE dbo.foo
(
    id INT NOT NULL
    , code CHAR(2) NOT NULL
);
```

## Tips

💡 To manage variable-length columns, the DBMS stores additional 2-byte information about the actual length. A VARCHAR(2) column  allocates twice as much disk space as a CHAR(2).
