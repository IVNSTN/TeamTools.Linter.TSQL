

# Fixed-length column with big size

|||
|-|-|
| Id | **DD0750**
| Mnemo | LONG_FIXED_LENGTH_COL
| Severity | ℹ Hint
| Category | [DatabaseDesign](./Category_DatabaseDesign.md)
| Source code | [LongFixedLengthColRule.cs](../../../Rules/DatabaseDesign/LongFixedLengthColRule.cs)

## Cause

This rule is triggered when fixed-length column with big size is used.
<p id="descr">Use variable-length column.</p>

## Examples

Bad

```sql
CREATE TABLE dbo.foo
(
    id INT NOT NULL
    , description CHAR(4000) NOT NULL
);
```

Good

```sql
CREATE TABLE dbo.foo
(
    id INT NOT NULL
    , description VARCHAR(4000) NOT NULL
);
```

## Tips

💡 Fixed-size columns store the full size specified in the column definition, regardless of the actual length of the row. If the column contains many short rows or the row size varies significantly, it is recommended to change the type from CHAR to VARCHAR.
