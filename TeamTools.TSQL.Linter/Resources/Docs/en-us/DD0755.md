

# Small column allows NULL

|||
|-|-|
| Id | **DD0755**
| Mnemo | SMALL_SIZE_COL_NULL
| Severity | ℹ Hint
| Category | [DatabaseDesign](./Category_DatabaseDesign.md)
| Source code | [SmallSizeColumnNullableRule.cs](../../../Rules/DatabaseDesign/SmallSizeColumnNullableRule.cs)

## Cause

<p id="descr">Для столбцов таблиц выбирают небольшие типы данных, в том числе, чтобы сэкономить место при хранении этих данных. Но если в такой колонке разрешить значения NULL, то СУБД будет выделять дополнительные биты (округлённые до целых байтов) для обозначения NULL-значений в каждой строке. Этих накладных расходов на учёт и обработку значений NULL можно избежать, если сделать колонки NOT NULL. Кроме прочего, ограниченные типы данных (BIT, TINYINT, короткие CHAR) часто используются как enum (перечисления) и физические ограничения типа важны для реализации идеи разработчика, а NULL в таком случае слишком особенное значение. В подобных ситуациях нередко можно обойтись без значения NULL что дополнительно сэкономит место и упростит логику в коде, обрабатывающем значения из подобных колонок. Если допустимо, то лучше сделать колонку NOT NULL.</p>
