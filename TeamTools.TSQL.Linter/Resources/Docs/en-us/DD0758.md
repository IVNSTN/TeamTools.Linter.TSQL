

# Redundant use of the LOCK_ESCALATION option

|||
|-|-|
| Id | **DD0758**
| Mnemo | REDUNDANT_TABLE_LOCK_ESCALATION
| Severity | ⚠ Warning
| Category | [DatabaseDesign](./Category_DatabaseDesign.md)
| Source code | [RedundantLockEscalationSetRule.cs](../../../Rules/DatabaseDesign/RedundantLockEscalationSetRule.cs)

## Cause

This rule is triggered when use of LOCK_ESCALATION option is redundant.
<p id="descr">Remove the LOCK_ESCALATION option.</p>

## Examples

Bad

```sql
CREATE TABLE dbo.foo
(
    id      INT  NOT NULL,
    date    DATE NOT NULL
);
GO

ALTER TABLE dbo.foo SET (LOCK_ESCALATION = TABLE);
GO
```

Good

```sql
CREATE TABLE dbo.foo
(
    id      INT  NOT NULL,
    date    DATE NOT NULL
);
GO

ALTER TABLE dbo.foo SET (LOCK_ESCALATION = DISABLE);
GO
```
