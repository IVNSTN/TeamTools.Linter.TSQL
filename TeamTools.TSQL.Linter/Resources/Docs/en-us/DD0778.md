

# SPARSE columns don't dominate the table

|||
|-|-|
| Id | **DD0778**
| Mnemo | SPARSE_AND_NOT_SPARSE
| Severity | ℹ Hint
| Category | [DatabaseDesign](./Category_DatabaseDesign.md)
| Source code | [SparseColumnsAsSecondaryRule.cs](../../../Rules/DatabaseDesign/SparseColumnsAsSecondaryRule.cs)

## Cause

This rule is triggered when the number of SPARSE columns is smaller than normal.
<p id="descr">Remove SPARSE option or move SPARSE columns to a separate table.</p>

## Examples

Bad

```sql
CREATE TABLE dbo.foo
(
    id           INT  NOT NULL
    , date       DATE NOT NULL DEFAULT GETDATE()
    , code       INT  NULL
    , is_enabled BIT  SPARSE NULL
);
```

Good

```sql
CREATE TABLE dbo.foo
(
    id           INT  NOT NULL
    , date       DATE NOT NULL DEFAULT GETDATE()
    , code       INT  NULL
    , is_enabled BIT  NULL
);
```
