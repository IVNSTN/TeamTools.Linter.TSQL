
# Short text field allows unicode

|||
|-|-|
| Id | **DD0824**
| Mnemo | SMALL_SIZE_UNICODE
| Severity | ℹ Hint
| Category | [DatabaseDesign](./Category_DatabaseDesign.md)
| Source code | [ForeignKeyRecursionRule.cs](../../../Rules/DatabaseDesign/SmallSizeColumnUnicodeRule.cs)

## Cause

This rule is triggered when a column with the `NVARCHAR` or `NCHAR` datatype and a length of less than 12 characters is detected.
<p id= "descr">Increase the length of the column or remove unicode support.</p>

## Examples

Bad

```sql
CREATE TABLE dbo.foo
(
    id        INT         NOT NULL
    , title   NVARCHAR(8) NOT NULL
)
```

Good

```sql
CREATE TABLE dbo.foo
(
    id        INT            NOT NULL
    , title   NVARCHAR(100)  NOT NULL
)

-- or
CREATE TABLE dbo.foo
(
    id        INT         NOT NULL
    , title   VARCHAR(8)  NOT NULL
)
```

## Tips

💡 A string column that supports unicode and can only store very short strings is suspicious. These columns are usually used to store mnemocodes, abbreviations, and enum-like values which do not require unicode.
