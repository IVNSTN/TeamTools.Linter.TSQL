

# History table not on the same schema as the origin

|||
|-|-|
| Id | **DD0831**
| Mnemo | HISTORY_IN_SAME_SCHEMA
| Severity | ℹ Hint
| Category | [DatabaseDesign](./Category_DatabaseDesign.md)
| Source code | [TemporalTableSameSchemaRule.cs](../../../Rules/DatabaseDesign/TemporalTableSameSchemaRule.cs)

## Cause

This rule is triggered when the history part of the temporal table belongs to a different schema.
<p id="descr">Specify the same schemas.</p>

## Examples

Bad

```sql
CREATE TABLE SchemaOne.Person
(
    ParsonID    INT          NOT NULL PRIMARY KEY CLUSTERED
    , FirstName VARCHAR(128) NOT NULL
    , LastName  VARCHAR(128) NOT NULL
    , BirthDate DATE
)
WITH (SYSTEM_VERSIONING = ON (HISTORY_TABLE = SchemaTwo.PersonHistory));
```

Good

```sql
CREATE TABLE SchemaOne.Person
(
    ParsonID    INT          NOT NULL PRIMARY KEY CLUSTERED
    , FirstName VARCHAR(128) NOT NULL
    , LastName  VARCHAR(128) NOT NULL
    , BirthDate DATE
)
WITH (SYSTEM_VERSIONING = ON (HISTORY_TABLE = SchemaOne.PersonHistory));
```

## Tips

💡 Usually it is not needed to create the history part of a temporal table in another schema, please check it.
