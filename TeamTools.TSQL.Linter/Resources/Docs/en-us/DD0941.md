
# Table is missing a primary key

|||
|-|-|
| Id | **DD0941**
| Mnemo | TABLE_HAS_PK
| Severity | ℹ Hint
| Category | [DatabaseDesign](./Category_DatabaseDesign.md)
| Source code | [TableHasPrimaryKeyRule.cs](../../../Rules/DatabaseDesign/TableHasPrimaryKeyRule.cs)

## Cause

This rule is triggered when the table is missing a primary key.
<p id="descr">Add primary key.</p>

## Examples

Bad

```sql
CREATE TABLE dbo.foo
(
    id          INT NOT NULL
    , parent_id INT NULL
    , name      VARCHAR(10) NOT NULL
);
```

Good

```sql
CREATE TABLE dbo.foo
(
    id          INT NOT NULL PRIMARY KEY
    , parent_id INT NULL
    , name      VARCHAR(10) NOT NULL
);
```

## Tips

💡 It is also highly desirable to set a primary key for temporary tables and table variables (# and @). Defining the primary key of a table is the same declaration of intent as determining whether a column is `NULL` or not.
