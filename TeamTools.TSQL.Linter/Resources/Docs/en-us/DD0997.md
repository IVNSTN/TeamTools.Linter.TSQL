

# Type is not recommended for keys

|||
|-|-|
| Id | **DD0997**
| Mnemo | BAD_TYPE_FOR_KEY
| Severity | ℹ Hint
| Category | [DatabaseDesign](./Category_DatabaseDesign.md)
| Source code | [BadTypeForKeyRule.cs](../../../Rules/DatabaseDesign/BadTypeForKeyRule.cs)

## Cause

This rule is triggered when the key column type is a large string or a number with a decimal part.
<p id="descr">Use other types of columns for the key.</p>

## Examples

Bad

```sql
CREATE TABLE dbo.foo
(
    id        VARCHAR(MAX) PRIMARY KEY 
    , type_id FLOAT FOREIGN KEY REFERENCES dbo.foo_types(type_id)
)
```

Good

```sql
CREATE TABLE dbo.foo
(
    id        VARCHAR(20) PRIMARY KEY 
    , type_id INT FOREIGN KEY REFERENCES dbo.foo_types(type_id)
)
```

## Tips

💡 The rule ignores COLUMNSTORE indexes.
