

# Type is not recommended for clustered index

|||
|-|-|
| Id | **DD0998**
| Mnemo | BAD_TYPE_FOR_CLUSTERED_IDX
| Severity | ℹ Hint
| Category | [DatabaseDesign](./Category_DatabaseDesign.md), [Indices](./Group_Indices.md)
| Source code | [BadTypeForClusteredIndexRule.cs](../../../Rules/DatabaseDesign/BadTypeForClusteredIndexRule.cs)

## Cause

This rule is triggered when clustered index column type is a large string or a number with a decimal part.
<p id="descr">Use other types of columns to include in the clustered index.</p>

## Examples

Bad

```sql
CREATE TABLE dbo.foo
(
    id      VARCHAR(MAX) PRIMARY KEY 
    , value FLOAT
)
```

Good

```sql
CREATE TABLE dbo.foo
(
    id      VARCHAR(20) PRIMARY KEY 
    , value FLOAT
)
```

## Tips

💡 The rule ignores COLUMNSTORE indexes.

💡 Editing columns included in an index moves rows from one index page to another, which can lead to performance degradation.
