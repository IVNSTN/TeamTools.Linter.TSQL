
# Target column count does not match source

|||
|-|-|
| Id | **FA0113**
| Mnemo | INSERT_COL_COUNT
| Severity | ⛔ Error
| Category | [Failure](./Category_Failure.md)
| Source code | [InsertColumnCountRule.cs](../../../Rules/Failure/InsertColumnCountRule.cs)

## Cause

This rule is triggered if the number of columns in the source does not match the number of columns in the target for the `MERGE`, `INSERT SELECT` or `OUTPUT INTO` operations.
<p id="descr">Align the number of columns between the source and target.</p>

## Examples

Bad

```sql
MERGE dbo.foo AS dst
USING dbo.bar AS src
ON src.id = dst.id
WHEN NOT MATCHED BY TARGET THEN
    INSERT (aa, bb, cc)
    VALUES (a, b);
```

Good

```sql
MERGE dbo.foo AS dst
USING dbo.bar AS src
ON src.id = dst.id
WHEN NOT MATCHED BY TARGET THEN
    INSERT (aa, bb, cc)
    VALUES (a, b, c);
```
