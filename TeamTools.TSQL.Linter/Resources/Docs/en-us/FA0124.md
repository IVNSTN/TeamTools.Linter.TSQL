
# Fetched values count does not match cursor columns count

|||
|-|-|
| Id | **FA0124**
| Mnemo | CURSOR_FETCH_COL_COUNT
| Severity | ⛔ Error
| Category | [Failure](./Category_Failure.md), [Cursors](./Group_Cursors.md)
| Source code | [CursorFetchColCountRule.cs](../../../Rules/Failure/CursorFetchColCountRule.cs)

## Cause

This rule is triggered if the number of variables filled in when reading from the cursor does not match the number of columns in the cursor definition.
<p id="descr">Align the number of fetched values with the number of cursor columns.</p>

## Examples

Bad

```sql
DECLARE cr CURSOR FAST_FORWARD FOR
    SELECT a, b, c FROM foo;
...
FETCH NEXT FROM cr
INTO
    @a
    , @b;
...
```

Good

```sql
DECLARE cr CURSOR FAST_FORWARD FOR
    SELECT a, b, c FROM foo;
...
FETCH NEXT FROM cr
INTO
    @a
    , @b
    , @c;
...
```
