
# Column number in UNION/EXCEPT/INTERSECT parts does not match

|||
|-|-|
| Id | **FA0132**
| Mnemo | UNION_COL_COUNT
| Severity | ⛔ Error
| Category | [Failure](./Category_Failure.md)
| Source code | [UnionColumnCountRule.cs](../../../Rules/Failure/UnionColumnCountRule.cs)

## Cause

This rule is triggered if the number of columns in the `UNION`, `EXCEPT`, or `INTERSECT` subqueries differs.
<p id="descr">Align the number of columns returned by subqueries.</p>

## Examples

Bad

```sql
SELECT a, b, c FROM dbo.table
INTERSECT
SELECT b, c, d, e FROM dbo.another_table;
```

Good

```sql
SELECT a, b, c, d FROM dbo.table
INTERSECT
SELECT b, c, d, e FROM dbo.another_table;
```

## Tips

💡 The number of columns must be the same in all subqueries of the `UNION`, `EXCEPT` or `INTERSECT` expression. If there is no data for any of the columns in the subquery, the column with the value `NULL` must still be explicitly declared in the selection.
