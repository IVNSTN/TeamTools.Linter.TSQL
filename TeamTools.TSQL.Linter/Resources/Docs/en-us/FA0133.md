
# Function argument number mismatch

|||
|-|-|
| Id | **FA0133**
| Mnemo | BUILTINS_ARG_COUNT
| Severity | ⛔ Error
| Category | [Failure](./Category_Failure.md)
| Source code | [BuiltInFunctionsArgumentCountRule.cs](../../../Rules/Failure/BuiltInFunctionsArgumentCountRule.cs)

## Cause

This rule is triggered if the wrong number of arguments is specified in a called function.
<p id="descr">Pass the correct number of arguments.</p>

## Examples

Bad

```sql
DECLARE @str VARCHAR(10);

SELECT 1
WHERE 1=1 AND (1=0 OR
    LOWER() IS NULL OR
    UPPER('asdf', 'asdf') IS NULL OR
    LEFT(@str) IS NULL OR
    REPLACE(@str, 'asdf') IS NULL OR
    ISNULL(@str) IS NULL OR
    DATEADD(DAY, '1990101') IS NULL OR
    DATEADD(DAY, '1990101', 3, 3) IS NULL)
```

Good

```sql
DECLARE @str VARCHAR(10);

SELECT 1
WHERE 1=1 AND (1=0 OR
    LOWER('asdf') IS NULL OR
    UPPER('asdf') IS NULL OR
    LEFT(@str, 3) IS NULL OR
    REPLACE(@str, 'asdf', 'fdsa') IS NULL OR
    ISNULL(@str, 'asdf') IS NULL OR
    DATEADD(DAY, 3, '1990101') IS NULL OR
```

## Tips

💡 The SSDT compiler skips expressions like `ISNULL(a.name )`, which the T-SQL compiler does not allow.

💡 Only simple built-in functions that have no variation in the set of arguments are checked.
