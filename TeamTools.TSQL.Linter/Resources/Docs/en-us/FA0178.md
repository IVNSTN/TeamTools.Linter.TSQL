
# Parameters definition in sp_executesql does not match passed params

|||
|-|-|
| Id | **FA0178**
| Mnemo | SP_EXECUTESQL_PARAMS_MISMATCH
| Severity | ⛔ Error
| Category | [Failure](./Category_Failure.md)
| Source code | [SpExecuteParamDefinitionMatchRule.cs](../../../Rules/Failure/SpExecuteParamDefinitionMatchRule.cs)

## Cause

This rule is triggered if the number of parameters or their names in the `sp_executesql` call do not match the way these parameters are declared for the dynamic code being called.
<p id="descr">Align the passed parameters to their definitions.</p>

## Examples

Bad

```sql
EXEC sp_executesql
    @stmt = N'select cmd'
    , @params = N'@date DATE, @sum MONEY NULL OUTPUT, @name CHAR(1)'
    , @sum = @sum
    , @date = @from_date;
```

Good

```sql
EXEC sp_executesql
    @stmt = N'select cmd'
    , @params = N'@date DATE, @sum MONEY NULL OUTPUT, @name CHAR(1)'
    , @name = 'test'
    , @sum = @sum OUTPUT
    , @date = @from_date;
```
