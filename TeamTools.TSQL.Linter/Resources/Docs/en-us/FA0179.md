

# Derived table column definition does not match query output

|||
|-|-|
| Id | **FA0179**
| Mnemo | OUTPUT_COL_DEFINITION_MISMATCH
| Severity | ⛔ Error
| Category | [Failure](./Category_Failure.md)
| Source code | [ParenthesisSubqueryDefinitionColCountRule.cs](../../../Rules/Failure/ParenthesisSubqueryDefinitionColCountRule.cs)

## Cause

<p id="descr">A subquery or CTE has a definition (<em>AS derived_query (col1, col2, ...)</em>) and it does not match the number of columns with what is written in the subquery itself.</p>
