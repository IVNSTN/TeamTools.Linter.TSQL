
# Derived table column definition does not match query output

|||
|-|-|
| Id | **FA0179**
| Mnemo | OUTPUT_COL_DEFINITION_MISMATCH
| Severity | ⛔ Error
| Category | [Failure](./Category_Failure.md)
| Source code | [ParenthesisSubqueryDefinitionColCountRule.cs](../../../Rules/Failure/ParenthesisSubqueryDefinitionColCountRule.cs)

## Cause

This rule is triggered if the number of columns in the CTE definition does not match the number of columns selected in the CTE query.
<p id="descr">Allign the column numbers of the CTE table definition with the subquery.</p>

## Examples

Bad

```sql
WITH cte (id, dt, name, summary) AS (SELECT 1, GETDATE())
SELECT * FROM cte;
```

Good

```sql
WITH cte (id, name, dt) AS (SELECT 1, NULL, GETDATE())
SELECT * FROM cte;
```
