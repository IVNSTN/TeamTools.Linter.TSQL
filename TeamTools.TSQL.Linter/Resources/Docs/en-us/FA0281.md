
# Subquery definition has duplicate column names

|||
|-|-|
| Id | **FA0281**
| Mnemo | UNIQUE_DERIVED_DEF_COL_NAME
| Severity | ⛔ Error
| Category | [Failure](./Category_Failure.md)
| Source code | [UniqueDerivedColumnAliasRule.cs](../../../Rules/Failure/UniqueDerivedColumnAliasRule.cs)

## Cause

This rule is triggered if the definition of a subquery or CTE contains duplicate column names.
<p id="descr">Set unique column names.</p>

## Examples

Bad

```sql
WITH cte (a, b, b) AS (SELECT 1, 2, 3)
SELECT * FROM cte
```

Good

```sql
WITH cte (a, b, c) AS (SELECT 1, 2, 3)
SELECT * FROM cte
```
