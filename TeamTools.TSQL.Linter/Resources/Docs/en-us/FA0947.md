

# Literal is out of range

|||
|-|-|
| Id | **FA0947**
| Mnemo | LITERAL_OUT_OF_RANGE
| Severity | ⛔ Error
| Category | [Failure](./Category_Failure.md)
| Source code | [LiteralOutOfRangeRule.cs](../../../Rules/Failure/LiteralOutOfRangeRule.cs)

## Cause

The rule is triggered if an integer literal is assigned to a variable whose size does not allow it to be stored.
<p id="descr">Specify the correct size for the variable.</p>

## Examples

Bad

```sql
DECLARE @var TINYINT;
SELECT ISNULL(@var, -1);

DECLARE @var BIT;
SELECT ISNULL(@var, -10);

DECLARE @var SMALLINT;
SELECT ISNULL(@var, 33000);
```

Good

```sql
DECLARE @var TINYINT;
SELECT ISNULL(@var, 255);

DECLARE @var BIT;
SELECT ISNULL(@var, 0);

DECLARE @var SMALLINT;
SELECT ISNULL(@var, 32767);
```

## Tips

💡 ISNULL truncates the second parameter to the size of the first one. The type and dimension of the ISNULL result are always equal to the type and dimension of the first parameter.

💡 Unlike ISNULL, COALESCE, calculates the final type, dimension, by taking into account all the specified arguments.
