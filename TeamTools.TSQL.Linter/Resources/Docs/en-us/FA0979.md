

# Insert modifies readonly column

|||
|-|-|
| Id | **FA0979**
| Mnemo | INSERT_MODIFIES_READONLY_COL
| Severity | ⚠ Warning
| Category | [Failure](./Category_Failure.md)
| Source code | [InsertModifiesReadonlyColumnRule.cs](../../../Rules/Failure/InsertModifiesReadonlyColumnRule.cs)

## Cause

The rule is triggered when trying to insert data into a computed or IDENTITY column.
<p id="descr">Remove the modification of the computed column or IDENTITY.</p>

## Examples

Bad

```sql
CREATE TABLE dbo.Product
(
    ProductId     INT          IDENTITY(1, 1) NOT NULL
    , ProductName VARCHAR(100) NOT NULL
    , Quantity    SMALLINT     NULL
    , Price       MONEY        NULL
    , Summa       AS (Quantity * Price) PERSISTED
);

INSERT INTO dbo.Product (ProductId, ProductName, Summa) 
VALUES (1, 'Product1', 100);
```

Good

```sql
CREATE TABLE dbo.Product
(
    ProductId     INT          IDENTITY(1, 1) NOT NULL
    , ProductName VARCHAR(100) NOT NULL
    , Quantity    SMALLINT     NULL
    , Price       MONEY        NULL
    , Summa       AS (Quantity * Price) PERSISTED
);

INSERT INTO dbo.Product (ProductName, Quantity, Price) 
VALUES ('Product1', 7, 100);
```

## Tips

💡 To insert a value into the IDENTITY column, you can use the statement SET IDENTITY_INSERT ON 

```sql
SET IDENTITY_INSERT dbo.Product ON;
INSERT INTO dbo.Product (ProductId, ProductName, Summa) 
VALUES (1, 'Product1', 7, 100);
SET IDENTITY_INSERT dbo.Product OFF;
```
