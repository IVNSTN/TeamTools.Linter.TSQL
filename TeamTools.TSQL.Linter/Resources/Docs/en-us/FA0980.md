

# Required column is missing from Insert specification

|||
|-|-|
| Id | **FA0980**
| Mnemo | INSERT_MISSES_REQUIRED_COL
| Severity | ⚠ Warning
| Category | [Failure](./Category_Failure.md)
| Source code | [InsertMissesRequiredColumnRule.cs](../../../Rules/Failure/InsertMissesRequiredColumnRule.cs)

## Cause

The rule is triggered if a column is declared as NOT NULL without a default value and is not specified in INSERT statement.
<p id="descr">Specify a value for the NOT NULL column.</p>

## Examples

Bad

```sql
CREATE TABLE dbo.Product
(
    ProductId     INT          NOT NULL
    , ProductName VARCHAR(100) NOT NULL DEFAULT ('product')
    , Quantity    SMALLINT     NULL
    , Price       MONEY        NULL
);

INSERT INTO dbo.Product (ProductName, Quantity, Price) 
VALUES ('Product1', 100, 5);
```

Good

```sql
CREATE TABLE dbo.Product
(
    ProductId     INT          NOT NULL
    , ProductName VARCHAR(100) NOT NULL DEFAULT ('product')
    , Quantity    SMALLINT     NULL
    , Price       MONEY        NULL
);

INSERT INTO dbo.Product (ProductId, ProductName, Quantity, Price) 
VALUES (1, 'Product1', 100, 5);
```

## Tips

💡 ROWVERSION, TIMESTAMP and IDENTITY columns are considered to have a "default".

💡 When IDENTITY_INSERT is ON, IDENTITY column is considered mandatory.
