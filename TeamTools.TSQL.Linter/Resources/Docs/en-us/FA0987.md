

# Duplicate column name is illegal in table definition

|||
|-|-|
| Id | **FA0987**
| Mnemo | DUP_COL_IN_TABLE
| Severity | ⛔ Error
| Category | [Failure](./Category_Failure.md)
| Source code | [DuplicateColInTableDefinitionRule.cs](../../../Rules/Failure/DuplicateColInTableDefinitionRule.cs)

## Cause

The rule is triggered if column name is not unique in a table definition.
<p id="descr">Come up with a unique column name.</p>

## Examples

Bad

```sql
CREATE TABLE dbo.Product
(
    ProductId     INT          IDENTITY(1, 1) NOT NULL
    , ProductName VARCHAR(100) NOT NULL
    , ProductType VARCHAR(100) NOT NULL
    , ProductName VARCHAR(100) NOT NULL
);
```

Good

```sql
CREATE TABLE dbo.Product
(
    ProductId     INT          IDENTITY(1, 1) NOT NULL
    , ProductName VARCHAR(100) NOT NULL
    , ProductType VARCHAR(100) NOT NULL
);
```

## Tips

💡 Remove duplicate columns from the table definition.
