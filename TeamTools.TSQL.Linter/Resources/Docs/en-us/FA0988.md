

# Invalid type size

|||
|-|-|
| Id | **FA0988**
| Mnemo | INVALID_TYPE_SIZE
| Severity | ⛔ Error
| Category | [Failure](./Category_Failure.md)
| Source code | [InvalidTypeSizeRule.cs](../../../Rules/Failure/InvalidTypeSizeRule.cs)

## Cause

This rule is triggered when the specified size is not supported by the data type.
<p id="descr">Specify the correct size of the data type.</p>

## Examples

Bad

```sql
DECLARE @tinyint TINYINT 
SET @tinyint = -1
SET @tinyint = 256

DECLARE @smallint SMALLINT
SET @smallint = -32769 
SET @smallint = 32768
```

Good

```sql
DECLARE @tinyint TINYINT
SET @tinyint = 0
SET @tinyint = 255

DECLARE @smallint SMALLINT 
SET @smallint = -32768
SET @smallint = 32767
```

## Tips

💡 A common mistake is that when changing the type from VARCHAR to NVARCHAR, the size remains 8000, although NVARCHAR allows no more than 4000 characters (except for the declaration with the MAX size).
    
💡 Another example: an error may occur when generating a dynamic SQL script if the length of the local variable used is more than 8000 characters in the case of the VARCHAR data type. Solution: in this case, you need to use the VARCHAR(MAX) data type.
