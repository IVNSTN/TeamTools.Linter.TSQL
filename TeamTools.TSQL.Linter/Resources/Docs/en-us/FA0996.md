
# Statement must be of type NVARCHAR or NCHAR

|||
|-|-|
| Id | **FA0996**
| Mnemo | SP_EXECUTESQL_NO_ASCII
| Severity | ⛔ Error
| Category | [Failure](./Category_Failure.md)
| Source code | [SpExecuteSqlNoAsciiSupportRule.cs](../../../Rules/Failure/SpExecuteSqlNoAsciiSupportRule.cs)

## Cause

This rule is triggered if the first or second parameter in a `sp_executesql` procedure call is passed without specifying a unicode datatype.
<p id="descr">For the first two parameters of the sp_executesql procedure, use a unicode datatype.</p>

## Examples

Bad

```sql
DECLARE @args CHAR(100) = '@msg VARCHAR(100)';

EXEC sp_executesql 
    'PRINT @msg'    -- 1
    , @args         -- 2
    , @msg = 'OK';
```

Good

```sql
DECLARE @args NCHAR(100) = N'@msg VARCHAR(100)';

EXEC sp_executesql 
    N'PRINT @msg'
    , @args
    , @msg = 'OK';
```
