

# DML statement key elements not aligned similarly

|||
|-|-|
| Id | **FM0219**
| Mnemo | STATEMENT_ALIGN
| Severity | ⚠ Warning
| Category | [Formatting](./Category_Formatting.md)
| Source code | [StatementElementsAlignedRule.cs](../../../Rules/Formatting/StatementElementsAlignedRule.cs)

## Cause

The rule is triggered if the key elements of DML expressions are not aligned consistently and do not have the same left indentation.
<p id="descr">Align the key elements of DML expressions.</p>

## Examples

Bad

```sql
INSERT INTO Employees (FirstName, LastName, Age)
    VALUES ('John', 'Doe', 30)
```

Good

```sql
INSERT INTO Employees (FirstName, LastName, Age)
VALUES ('John', 'Doe', 30)
```
