

# Non-primary file group hardcoded

|||
|-|-|
| Id | **HD0915**
| Mnemo | NON_PRIMARY_FILEGROUP
| Severity | ⚠ Warning
| Category | [Hardcode](./Category_Hardcode.md), [Indices](./Group_Indices.md)
| Source code | [IndexFileGroupNonPrimaryRule.cs](../../../Rules/Hardcode/IndexFileGroupNonPrimaryRule.cs)

## Cause

This rule is triggered if an reference to a non-PRIMARY filegroup is found in the code.
<p id="descr">Replace the filegroup with PRIMARY</p>

## Examples

Bad

```sql
CREATE TABLE dbo.foo
(
    id INT
    , name VARCHAR(100)
    , CONSTRAINT PK_foo PRIMARY KEY CLUSTERED (id) ON [ARCHIVE]
) ON [ARCHIVE]

CREATE INDEX IX_foo ON dbo.foo(name)
ON [ARCHIVE]
```

Good

```sql
CREATE TABLE dbo.foo
(
    id INT
    , name VARCHAR(100)
    , CONSTRAINT PK_foo PRIMARY KEY CLUSTERED (id) ON [PRIMARY]
) ON [PRIMARY]

CREATE INDEX IX_foo ON dbo.foo(name)
ON [PRIMARY]
```

## Tips

💡File groups are the operational area, the area of ​​responsibility of the DBA. There is no need to "program" them.
