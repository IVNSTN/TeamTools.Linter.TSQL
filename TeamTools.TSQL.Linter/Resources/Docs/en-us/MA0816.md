

# Too many values for IN predicate

|||
|-|-|
| Id | **MA0816**
| Mnemo | IN_VALUES_TOO_MANY
| Severity | ℹ Hint
| Category | [Maintainablity](./Category_Maintainablity.md)
| Source code | [InPredicateTooManyValuesRule.cs](../../../Rules/Maintainability/InPredicateTooManyValuesRule.cs)

## Cause

This rule is triggered when the IN predicate contains more than 12 values.
<p id="descr">Put the values ​​into a table or table variable.</p>

## Examples

Bad

```sql
SELECT lastname
FROM dbo.users
WHERE firstname IN ('Adam', 'Adrian', 'Andrew', 'Arnold', 'Brian', 'Caleb', 'Carl', 'Chad', 'Daniel', 'Derek', 'Elliot', 'Eric', 'Ethan');
```

Good

```sql
DECLARE @names TABLE (name VARCHAR(20) NOT NULL);

INSERT INTO @names (name)
VALUES
('Adam')
, ('Adrian')
, ('Andrew')
, ('Arnold')
, ('Brian')
, ('Caleb')
, ('Carl')
, ('Chad')
, ('Daniel')
, ('Derek')
, ('Elliot')
, ('Eric')
, ('Ethan');

SELECT lastname FROM dbo.users WHERE firstname IN (SELECT name FROM @names);
```
