

# Constraint name violates naming pattern

|||
|-|-|
| Id | **NM0203**
| Mnemo | CONSTRAINT_NAME_PATTERN
| Severity | ⚠ Warning
| Category | [Naming](./Category_Naming.md)
| Source code | [ConstraintNameFormatRule.cs](../../../Rules/Naming/ConstraintNameFormatRule.cs)

## Cause

This rule is triggered when constraint name violates naming pattern.
<p id="descr">Make the constraint name match the naming pattern.</p>

## Examples

Bad

```sql
ALTER TABLE pak.test ADD CONSTRAINT PK_test PRIMARY KEY (id);
```

Good

```sql
ALTER TABLE pak.test ADD CONSTRAINT PK_pak_test PRIMARY KEY (id);
```

## Tips

💡 The rule is case-sensitive

💡 The rule applies to table and column constraints:

- PRIMARY KEY
- FOREIGN KEY
- DEFAULT
- UNIQUE
- CHECK
