

# Variable name spelling differs from declaration

|||
|-|-|
| Id | **NM0204**
| Mnemo | VAR_NAME_MISSPELLED
| Severity | ⚠ Warning
| Category | [Naming](./Category_Naming.md)
| Source code | [VariableNameSpellingRule.cs](../../../Rules/Naming/VariableNameSpellingRule.cs)

## Cause

This rule is triggered when variable name spelling differs from declaration.
<p id="descr">Unify the variable name.</p>

## Examples

Bad

```sql
DECLARE @VarOne INT
SET @varone = 1;
```

Good

```sql
DECLARE @VarOne INT
SET @VarOne = 1;
```

## Tips

💡 Use "Format SQL" after renaming the variable in DECLARE section to edit a variable name case (Replace command is unsafe).

💡 Use Refactoring/Rename command for correctly rename a variable.
