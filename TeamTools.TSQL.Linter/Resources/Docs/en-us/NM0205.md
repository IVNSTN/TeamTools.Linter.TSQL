

# Mix of variable naming notations

|||
|-|-|
| Id | **NM0205**
| Mnemo | VAR_NAME_NOTATION_MIX
| Severity | ℹ Hint
| Category | [Naming](./Category_Naming.md)
| Source code | [VariableNamingNotationRule.cs](../../../Rules/Naming/VariableNamingNotationRule.cs)

## Cause

This rule is triggered if mix of variable naming conventions (PascalCase, snake_case) is found in the code.
<p id="descr">Use a single naming convention.</p>

## Examples

Bad

```sql
DECLARE @first_number INT
DECLARE @SeconNumber INT
```

Good

```sql
DECLARE @FirstNumber INT
DECLARE @SeconNumber INT
```

## Tips

💡 If you are not sure check Readme.md in repository. If you can't rename arguments for stored procedure, you can create variables in a correct convention, assign arguments to variables and use variables.
