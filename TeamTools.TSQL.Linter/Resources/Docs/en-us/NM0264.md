

# Unreadable identifier

|||
|-|-|
| Id | **NM0264**
| Mnemo | UNREADABLE_IDENTIFIER
| Severity | ℹ Hint
| Category | [Naming](./Category_Naming.md)
| Source code | [UnreadableIdentifierRule.cs](../../../Rules/Naming/UnreadableIdentifierRule.cs)

## Cause

This rule is triggered if unreadable identifier (only numbers, single-symbol etc.) is found in the code.
<p id="descr">Use readable identifier.</p>

## Examples

Bad

```sql
DECLARE @f123 INT
CREATE TABLE [77t-@] ([x0f-9 ] INT)
```

Good

```sql
DECLARE @foo INT
DECLARE @i INT
```

## Tips

💡 In the translation or ETL code you have to give up the external names as early as possible.

💡 The rule ignores @i, @j abbreviations.
