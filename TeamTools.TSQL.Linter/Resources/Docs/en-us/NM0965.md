

# Table naming convention violation - PascalCase expected

|||
|-|-|
| Id | **NM0965**
| Mnemo | TABLE_NAME_PASCAL_CASE
| Severity | ℹ Hint
| Category | [Naming](./Category_Naming.md)
| Source code | [TableViewNamePascalCaseRule.cs](../../../Rules/Naming/TableViewNamePascalCaseRule.cs)

## Cause
The rule is triggered if the project settings specify
that the table name must match the PascalCase notation,
but it does not match.

<p id="descr">Use PascalCase notation.</p>

## Examples

Bad

```sql
CREATE TABLE dbo.reg_customers (id BIGINT, name NVARCHAR(50));
```

Good

```sql
CREATE TABLE dbo.RegCustomers (id BIGINT, name NVARCHAR(50));
```

## Tips

💡 The rule also applies to temporary tables # and table variables @

💡 Requires such notation only in projects where it is explicitly specified in the settings.
