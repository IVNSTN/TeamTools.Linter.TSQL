

# Service broker naming convention violation - lower_snake_case expected

|||
|-|-|
| Id | **NM0971**
| Mnemo | SB_NAME_LOWER_SNAKE_CASE
| Severity | ℹ Hint
| Category | [Naming](./Category_Naming.md)
| Source code | [ServiceBrokerPublicNameLowerSnakeCaseRule.cs](../../../Rules/Naming/ServiceBrokerPublicNameLowerSnakeCaseRule.cs)

## Cause

The rule is triggered if the project settings specify
that the Servoce Broker variables name must match the lower_snake_case notation,
but it does not match.

<p id="descr">Use lower_snake_case notation.</p>

## Examples

Bad

```sql
DECLARE @myVar NVARCHAR(50);
```

Good

```sql
DECLARE @my_var NVARCHAR(50);
```

## Tips

💡 Requires such notation only in projects where it is explicitly specified in the settings.
