

# Variable naming convention violation - PascalCase expected

|||
|-|-|
| Id | **NM0973**
| Mnemo | VARIABLE_NAME_PASCAL_CASE
| Severity | ℹ Hint
| Category | [Naming](./Category_Naming.md)
| Source code | [VariableNamePascalCaseRule.cs](../../../Rules/Naming/VariableNamePascalCaseRule.cs)

## Cause

The rule is triggered if the project settings specify
that the variable name must match the PascalCase notation,
but it does not match.

<p id="descr">Use PascalCase notation.</p>

## Examples

Bad

```sql
DECLARE @my_var NVARCHAR(50);
```

Good

```sql
DECLARE @MyVar NVARCHAR(50);
```

## Tips

💡 Table variables follow the rules for tables.

💡 Requires such notation only in projects where it is explicitly specified in the settings.
