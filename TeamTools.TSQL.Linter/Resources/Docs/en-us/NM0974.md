

# Variable naming convention violation - lower_snake_case expected

|||
|-|-|
| Id | **NM0974**
| Mnemo | VARIABLE_NAME_LOWER_SNAKE_CASE
| Severity | ℹ Hint
| Category | [Naming](./Category_Naming.md)
| Source code | [VariableNameLowerSnakeCaseRule.cs](../../../Rules/Naming/VariableNameLowerSnakeCaseRule.cs)

## Cause

The rule is triggered if the project settings specify
that the variable name must match the lower_snake_case notation,
but it does not match.

<p id="descr">Use lower_snake_case notation.</p>

## Examples

Bad

```sql
DECLARE @myVar NVARCHAR(50);
```

Good

```sql
DECLARE @my_var NVARCHAR(50);
```

## Tips

💡 Table variables follow the rules for tables.

💡 Requires such notation only in projects where it is explicitly specified in the settings.
