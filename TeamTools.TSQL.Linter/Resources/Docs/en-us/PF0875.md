

# DDL is mixed with DML

|||
|-|-|
| Id | **PF0875**
| Mnemo | DDL_DML_MIX
| Severity | ℹ Hint
| Category | [Performance](./Category_Performance.md)
| Source code | [MixOfDdlAndDmlRule.cs](../../../Rules/Performance/MixOfDdlAndDmlRule.cs)

## Cause

This rule is triggered when DDL is mixed with DML.
<p id="descr">Move all DDL commands to the beginning of the query.</p>

## Examples

Bad

```sql
CREATE PROC dbo.delete_data
AS
BEGIN
    CREATE TABLE #t (id INT);

    SELECT br.id FROM dbo.bar AS br INNER JOIN #t AS tt ON tt.id = br.parent_id;

    CREATE INDEX ix ON #t (id);

    DELETE bar FROM dbo.bar AS br INNER JOIN #t AS tt ON tt.id = br.parent_id;
END;
GO
```

Good

```sql
CREATE PROC dbo.delete_data
AS
BEGIN
    CREATE TABLE #t (id INT);

    CREATE INDEX ix ON #t (id);

    SELECT br.id FROM dbo.bar AS br INNER JOIN #t AS tt ON tt.id = br.parent_id;

    DELETE bar FROM dbo.bar AS br INNER JOIN #t AS tt ON tt.id = br.parent_id;
END;
GO
```

## Tips

💡 This rule ignores queries without a source such as STRING_SPLIT, OPENJSON, OPENXML and manipulations of table variables.
