
# Redundant DEFAULT NULL constraint

|||
|-|-|
| Id | **RD0185**
| Mnemo | DEFAULT_NULL
| Severity | ⚠ Warning
| Category | [Redundancy](./Category_Redundancy.md)
| Source code | [NullDefaultNullRule.cs](../../../Rules/Redundancy/NullDefaultNullRule.cs)

## Cause

This rule is triggered if the default value `NULL` is assigned to a `NOT NULL` or NULLable column.
<p id="descr">Delete the default with 'NULL' value.</p>

## Examples

Bad

```sql
CREATE TABLE dbo.employee(id INT NOT NULL, salary INT NULL DEFAULT NULL)
```

Good

```sql
CREATE TABLE dbo.employee(id INT NOT NULL, salary INT NULL)
```

## Tips

💡 There is a chance that the `DEFAULT` keyword is used somewhere when assigning this column. Then, when this `DEFAULT` is dropped, such a code will break. However, this is a rare scenario. `NULL` can be inserted explicitly, or the column can be removed from the `INSERT` - `NULL` will be automatically assigned. It is best to correct the code itself and remove unnecessary `DEFAULT`.
