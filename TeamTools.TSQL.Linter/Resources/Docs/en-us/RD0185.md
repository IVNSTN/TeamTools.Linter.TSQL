

# Redundant DEFAULT NULL constraint

|||
|-|-|
| Id | **RD0185**
| Mnemo | DEFAULT_NULL
| Severity | ⚠ Warning
| Category | [Redundancy](./Category_Redundancy.md)
| Source code | [NullDefaultNullRule.cs](../../../Rules/Redundancy/NullDefaultNullRule.cs)

## Cause

<p id="descr">For NOT NULL columns, choosing NULL as the default is an error, for NULLable it is unnecessary. NULL will insert itself quite successfully.</p>

## Examples

Bad

```sql
CREATE TABLE dbo.employee(id INT NOT NULL, salary INT NULL DEFAULT NULL)
```

Good

```sql
CREATE TABLE dbo.employee(id INT NOT NULL, salary INT NULL)
```

## Tips

💡 There is a chance that the DEFAULT keyword is used somewhere when filling in this field, then when the default is demolished, such a code will break. But this is a strange scenario. NULL can be explicitly inserted, and the field can be removed from the insert altogether - NULL will turn out by itself. For the sake of such a code, you do not need to save the default, it is better to correct the code itself, based on an unnecessary default.
