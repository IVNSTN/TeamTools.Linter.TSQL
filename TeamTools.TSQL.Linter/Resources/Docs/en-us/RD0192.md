
# Pointless attempt to control EXISTS output

|||
|-|-|
| Id | **RD0192**
| Mnemo | EXISTS_OUTPUT_CONTROL
| Severity | ⚠ Warning
| Category | [Redundancy](./Category_Redundancy.md)
| Source code | [ExistsOutputControlRule.cs](../../../Rules/Redundancy/ExistsOutputControlRule.cs)

## Cause

This rule is triggered if the `EXISTS` subquery returns a list of columns, or when is uses `TOP`, `ORDER BY`, `OFFSET` or `GROUP BY` without `HAVING` clause.
<p id="descr">Replace the list of column with 'SELECT 1' and remove any unnecessary instructions.</p>

## Examples

Bad

```sql
IF EXISTS (SELECT TOP(10) id FROM dbo.employee ORDER BY salary)...
```

Good

```sql
IF EXISTS (SELECT 1 FROM dbo.employee)...
```

## Tips

💡 When processing `EXISTS`, the DB Engine completely ignores the `SELECT` part, so neither columns nor `TOP` are needed. It will not be able to return the multiple rows either, so it does not need `ORDER BY`, `GROUP BY` (without `HAVING`), `OFFSET` clauses.
