

# Redundant negation

|||
|-|-|
| Id | **RD0286**
| Mnemo | REDUNDANT_NEGATION
| Severity | ⚠ Warning
| Category | [Redundancy](./Category_Redundancy.md)
| Source code | [RedundantNegationRule.cs](../../../Rules/Redundancy/RedundantNegationRule.cs)

## Cause

<p id="descr">There is no need to negate the comparison operators from the outside, because you can just use the opposite operator.</p>

## Examples

Bad

```sql
IF NOT (@a = @b)
BEGIN
...
END;
```

Good

```sql
IF @a <> @b
BEGIN
...
END;
```

## Tips

💡 With ANSI_NULLS enabled, any option to compare something with NULL without using IS gives UNKNOWN, negating UNKNOWN or inverting the comparison gives the same UNKNOWN.
