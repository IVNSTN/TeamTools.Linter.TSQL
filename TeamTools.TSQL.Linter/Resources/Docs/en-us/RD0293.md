

# Unnecessary ELSE NULL option

|||
|-|-|
| Id | **RD0293**
| Mnemo | REDUNDANT_CASE_ELSE_NULL
| Severity | ⚠ Warning
| Category | [Redundancy](./Category_Redundancy.md)
| Source code | [RedundantCaseElseNullRule.cs](../../../Rules/Redundancy/RedundantCaseElseNullRule.cs)

## Cause

<p id="descr">You do not need to explicitly specify the ELSE NULL option in the CASE block - NULL will be returned if none of the options fit.</p>

## Examples

Bad

```sql
SELECT CASE WHEN @a = 5 THEN 4 ELSE NULL END;
```

Good

```sql
SELECT CASE WHEN @a = 5 THEN 4 END;
```

## Tips

💡 There is also an opposite point of view that all options must be explicitly declared. Both have an equally precarious position, the choice is made in favor of one particular one.
