
# Expand scalar predicate, using EXISTS here makes no point

|||
|-|-|
| Id | **RD0719**
| Mnemo | SCALAR_PREDICATE_AS_EXISTS
| Severity | ⚠ Warning
| Category | [Redundancy](./Category_Redundancy.md)
| Source code | [ScalarPredicateTurnedIntoExistsRule.cs](../../../Rules/Redundancy/ScalarPredicateTurnedIntoExistsRule.cs)

## Cause

This rule is triggered if the predicate `EXISTS` is used without the `FROM` construction, wrapping a boolean expression in a subquery.
<p id="descr">Remove EXISTS by moving its internal WHERE to an external expression.</p>

## Examples

Bad

```sql
IF EXISTS(SELECT 1 WHERE @a > @b)
BEGIN
    ...
END
```

Good

```sql
IF @a > @b
BEGIN
    ...
END
```
