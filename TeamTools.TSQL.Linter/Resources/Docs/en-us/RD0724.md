

# Redundant index option

|||
|-|-|
| Id | **RD0724**
| Mnemo | REDUNDANT_INDEX_OPTION
| Severity | ℹ Hint
| Category | [Redundancy](./Category_Redundancy.md), [Indices](./Group_Indices.md)
| Source code | [RedundantIndexOptionRule.cs](../../../Rules/Redundancy/RedundantIndexOptionRule.cs)

## Cause

This rule is triggered if the index creation instructions specify option values ​​that match the default values.
<p id="descr">Remove the option setting with values ​​that match the default.</p>

## Examples

Bad

```sql
CREATE TABLE dbo.foo
(
    id INT,
    title VARCHAR(20),
    CONSTRAINT PK PRIMARY KEY CLUSTERED (id)
    WITH (PAD_INDEX = OFF, ALLOW_ROW_LOCKS = ON)
)

CREATE INDEX IX ON dbo.foo(title)
WITH (SORT_IN_TEMPDB = OFF, DROP_EXISTING = OFF)
```

Good

```sql
CREATE TABLE dbo.foo
(
    id INT,
    title VARCHAR(20),
    CONSTRAINT PK PRIMARY KEY CLUSTERED (id)
)

CREATE INDEX IX ON dbo.foo(title)
WITH (SORT_IN_TEMPDB = OFF)
```
