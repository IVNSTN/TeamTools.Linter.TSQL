
# Multiple equality checks can be combined into single IN predicate

|||
|-|-|
| Id | **SI0845**
| Mnemo | MULTIPLE_OR_TO_IN
| Severity | ℹ Hint
| Category | [Simplification](./Category_Simplification.md)
| Source code | [MultipleOrToInRule.cs](../../../Rules/Simplification/MultipleOrToInRule.cs)

## Cause

This rule is triggered when multiple equality checks can be combined into single `IN` predicate.
<p id="descr">Combine the checks into single IN predicate.</p>

## Examples

Bad

```sql
SELECT *
FROM dbo.foo
WHERE category_id IN (1, 2, 3)
    OR category_id = 4
    OR 5 = category_id;
```

Good

```sql
SELECT *
FROM dbo.foo
WHERE category_id IN (1, 2, 3, 4, 5);
```
