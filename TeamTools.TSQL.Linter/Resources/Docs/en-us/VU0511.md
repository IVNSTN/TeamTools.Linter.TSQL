

# Unexpected job management within code block

|||
|-|-|
| Id | **VU0511**
| Mnemo | JOB_MANAGEMENT
| Severity | ⚠ Warning
| Category | [Vulnerability](./Category_Vulnerability.md)
| Source code | [JobManagementInCodeRule.cs](../../../Rules/Vulnerability/JobManagementInCodeRule.cs)

## Cause

The rule is triggered when code contains job manajement commands.
<p id="descr">Remove commands for job management.</p>

## Examples

Bad

```sql
EXEC sp_start_job 'My Job';
```

## Tips

💡 Choose dedicated schemas and databases for internal and system procedures.

💡 With access to service procedures an ordinary user of the product can directly or indirectly gain access to the management of the DBMS, which is unsafe. Only DBAs can have this level of access.
