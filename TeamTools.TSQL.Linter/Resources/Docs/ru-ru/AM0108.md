

# Создание объекта без указания схемы

|||
|-|-|
| Id | **AM0108**
| Мнемо | CREATE_WITH_SCHEMA
| Серьёзность | ⚠ Предупреждение
| Категория | [Неоднозначность](./Категория_Неоднозначность.md)
| Исходный код | [SchemaQualifiedObjectCreationRule.cs](../../../Rules/Ambiguity/SchemaQualifiedObjectCreationRule.cs)

## Причина

<p id="descr">Правило срабатывает, если не указана схема в инструкции создания объекта.</p>

## Примеры

Некорректно

```sql
CREATE TABLE employee ...
CREATE VIEW hiredate_view ...
```

Корректно

```sql
CREATE TABLE dbo.employee ...
CREATE VIEW dbo.hiredate_view ...
```

## Подсказки

💡 Разные пользователи могут иметь разные DEFAULT_SCHEMA, за исключением членов роли sysadmin. Следует всегда указывать схему в скриптах, создающих объекты, потому что схема не должна зависеть от пользователя, который запустит скрипт.
