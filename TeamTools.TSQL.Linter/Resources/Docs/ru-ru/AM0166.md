

# Целевые имена столбцов не совпадают с именами источника

|||
|-|-|
| Id | **AM0166**
| Мнемо | INSERT_COLUMN_ALIAS_MATCH_TARGET
| Серьёзность | ⚠ Предупреждение
| Категория | [Неоднозначность](./Категория_Неоднозначность.md)
| Исходный код | [InsertedColumnAliasMatchTargetRule.cs](../../../Rules/Ambiguity/InsertedColumnAliasMatchTargetRule.cs)

## Причина

Правило срабатывает, если есть разница между именами столбцов источника и целевого объекта в INSERT-SELECT.
<p id="descr">Приведите в соответствие имена столбцов источника и целевого объекта.</p>

## Примеры

Некорректно

```sql
INSERT INTO dbo.employee (person_id, person_name, is_remote)
SELECT
    PN.id
    , PN.full_name
    , IIF(PN.planet = 'Земля', 0, 1) AS is_alien
FROM dbo.person AS PN;
```

Корректно

```sql
INSERT INTO dbo.employee (person_id, person_name, is_remote)
SELECT
    PN.id as person_id
    , PN.full_name as person_name
    , IIF(PN.planet = 'Земля', 0, 1) AS is_remote
FROM dbo.person AS PN;
```

## Подсказки

💡 Используйте алиасы для приведения имён столбцов источника к целевым именам в INSERT-SELECT, чтобы подтвердить корректность списка столбцов.
