

# Алиас таблицы или подзапроса совпадает с именем другой таблицы

|||
|-|-|
| Id | **AM0170**
| Мнемо | TABLE_ALIAS_MIMICKS_OTHER_TABLE
| Серьёзность | ⚠ Предупреждение
| Категория | [Неоднозначность](./Категория_Неоднозначность.md)
| Исходный код | [TableAliasMimicksOtherTableRule.cs](../../../Rules/Ambiguity/TableAliasMimicksOtherTableRule.cs)

## Причина

Правило срабатывает, если в запросе алиас, выданный таблице или подзапросу, совпадает с именем другой используемой таблицы.
<p id="descr">Задайте уникальный алиас.</p>

## Примеры

Некорректно

```sql
SELECT old.id AS name, new.name AS id
FROM dbo.tbl AS t
INNER JOIN t AS old
    ON 1=1
INNER JOIN old AS new
    ON 1=1
```

Корректно

```sql
SELECT old.id AS name, new.name AS id
FROM dbo.tbl AS t
INNER JOIN #new AS old
    ON 1=1
INNER JOIN #old AS new
    ON 1=1
```

## Подсказки

💡 Выбирайте отстраненные алиасы. Даже если вам нужно перевернуть конструкцию вида

```sql
FROM new as old
JOIN old as new
```

постарайтесь выбрать алиасы, не пересекающиеся с именами объектов.
