

# Вызов OBJECT_ID() без указания типа объекта

|||
|-|-|
| Id | **AM0718**
| Мнемо | OBJECT_ID_WITHOUT_TYPE
| Серьёзность | ⚠ Предупреждение
| Категория | [Неоднозначность](./Категория_Неоднозначность.md)
| Исходный код | [ObjectIdNeedsTypeRule.cs](../../../Rules/Ambiguity/ObjectIdNeedsTypeRule.cs)

## Причина

Правило срабатывает, если при вызове функции `OBJECT_ID` не указан тип объекта.
<p id="descr">Укажите тип объекта.</p>

## Примеры

Некорректно

```sql
SELECT OBJECT_ID('dbo.my_object')
```

Корректно

```sql
SELECT OBJECT_ID('dbo.my_object', 'U')
```

## Подсказки

💡 Функция `OBJECT_ID` может найти не тот, объект, что вы искали, если не указан тип объекта. Нужно указывать тип, например, для таблиц это будет `OBJECT_ID('dbo.my_table', 'U')`.

💡 Список типов объектов содержится в столбце `type` представления `sys.objects`.
