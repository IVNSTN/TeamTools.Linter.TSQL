

# Неоднозначность отрицания составного предиката

|||
|-|-|
| Id | **AM0728**
| Мнемо | NOT_FOR_COMPLEX_PREDICATE
| Серьёзность | ℹ Подсказка
| Категория | [Неоднозначность](./Категория_Неоднозначность.md)
| Исходный код | [ComplexPredicateNegationRule.cs](../../../Rules/Ambiguity/ComplexPredicateNegationRule.cs)

## Причина

<p id="descr">Правило срабатывает, если обнаружено отрицание составного предиката без явного определения области применения отрицания.</p>

## Примеры

Некорректно

```sql
IF NOT @a = 0 AND @a > b
    PRINT '1'
```

Корректно

```sql
IF (NOT @a IS NULL) AND @a > b
    PRINT '1'
```

## Подсказки

💡 Отрицание составного предиката может быть воспринято неоднозначно: один прочитает как отрицание первого выражения, другой - как отрицание всего сочетания выражений предиката. Лучше добавить пару скобок и сделать код более однозначным.

💡 Не реагирует на предикаты, в которых все составные фрагменты имеют свои отрицания: NOT &lt;expr&gt; AND NOT &lt;expr&gt; OR NOT &lt;expr&gt;...
