

# Не указан алиас таблицы-источника для колонки

|||
|-|-|
| Id | **AM0935**
| Мнемо | AMBIGUOUS_COL_SOURCE
| Серьёзность | ⚠ Предупреждение
| Категория | [Неоднозначность](./Категория_Неоднозначность.md)
| Исходный код | [AmbiguousColumnBelongingRule.cs](../../../Rules/Ambiguity/AmbiguousColumnBelongingRule.cs)

## Причина

<p id="descr">Правило срабатывает, если у колонки не указан алиас таблицы-источника.</p>

## Примеры

Некорректно

```sql
UPDATE t SET
    title = b.new_title
OUTPUT t.title
FROM dbo.foo AS t
INNER JOIN dbo.bar AS b
    ON t.id = parent_id
WHERE title LIKE 'asdf%'
GO
```

Корректно

```sql
UPDATE t SET
    title = b.new_title,
FROM dbo.foo AS t
INNER JOIN dbo.bar AS b
    ON t.id = b.parent_id
WHERE t.title LIKE 'asdf%';
```

## Подсказки

💡 Выдайте таблицам алиасы, проставьте алиасы всем упоминаемым колонкам.

💡 Даже если пишется запрос из одной таблицы, то лучше все равно выдать ей алиас и проставить этот алиас всем колонкам. Если этого не сделать, то получится "футбол" в сторону следующего разработчика, которому выпадет добавлять в этот запрос джойн. Тогда ему придется вместо добавления этого одного джойна сначала отрефачить весь старый запрос. Это не самый конструктивный сценарий.
