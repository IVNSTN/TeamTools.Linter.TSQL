

# Множественные изменения переменной в одном выражении

|||
|-|-|
| Id | **AM0996**
| Мнемо | MULTI_SET_SAME_VAR
| Серьёзность | ⚠ Предупреждение
| Категория | [Неоднозначность](./Категория_Неоднозначность.md)
| Исходный код | [MultipleVarAssignmentRule.cs](../../../Rules/Ambiguity/MultipleVarAssignmentRule.cs)

## Причина

Правило срабатывает при обнаружении множественных попыток присвоить значение переменной в рамках одного выражения.
<p id="descr">Используйте разные переменные.</p>

## Примеры

Некорректно

```sql
EXEC @res = dbo.test
    @param = @res OUTPUT

FETCH NEXT FROM @cursor
INTO @a, @a
```

Корректно

```sql
EXEC @res_code = dbo.test
    @param = @res OUTPUT

FETCH NEXT FROM @cursor
INTO @a, @b
```

## Подсказки

💡 Множественные попытки присвоить значение переменной в рамках одного выражения порождают неопределённость. Перепишите выражение так, чтобы каждая переменная получала новое значения единожды.
