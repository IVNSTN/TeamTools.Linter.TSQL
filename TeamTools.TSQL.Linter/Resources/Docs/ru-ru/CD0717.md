

# Выражение CREATE OR ALTER под запретом, используйте CREATE

|||
|-|-|
| Id | **CD0717**
| Мнемо | CREATE_OR_ALTER
| Серьёзность | ⛔ Ошибка
| Категория | [НепрерывнаяПоставка](./Категория_НепрерывнаяПоставка.md)
| Исходный код | [CreateOrAlterRule.cs](../../../Rules/ContinuousDeployment/CreateOrAlterRule.cs)

## Причина

Правило срабатывает, если определяемый в файле скрипта объект создается выражением `CREATE OR ALTER`.
<p id="descr">Используйте CREATE вместо CREATE OR ALTER.</p>

## Примеры

Некорректно

```sql
CREATE OR ALTER PROCEDURE foo
AS
BEGIN
    SET NOCOUNT ON;

    RETURN 0;
END;
GO
```

Корректно

```sql
CREATE PROCEDURE foo
AS
BEGIN
    SET NOCOUNT ON;

    RETURN 0;
END;
GO
```

## Подсказки

💡 Выражения `CREATE OR ALTER` абсолютно валидны, но не поддерживаются механизмами DacFx и плохо влияют на CI/CD.
