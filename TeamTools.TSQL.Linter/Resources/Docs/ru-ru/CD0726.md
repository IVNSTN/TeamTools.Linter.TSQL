

# Неожиданный блок кода в корне скрипта

|||
|-|-|
| Id | **CD0726**
| Мнемо | CODE_IN_SCRIPT_ROOT
| Серьёзность | ⚠ Предупреждение
| Категория | [НепрерывнаяПоставка](./Категория_НепрерывнаяПоставка.md)
| Исходный код | [RootScriptBlockRule.cs](../../../Rules/ContinuousDeployment/RootScriptBlockRule.cs)

## Причина

Правило срабатывает, если в корне скрипта определения объекта присутствует посторонний код.
<p id="descr">Уберите посторонний код из скрипта.</p>

## Примеры

Некорректно

```sql
CREATE TABLE foo
(
    bar INT NOT NULL CONSTRAINT cns DEFAULT 0
    , mar INT NOT NULL
)
GO

INSERT INTO foo (bar, mar)
VALUES (1, 2)
```

Корректно

```sql
CREATE TABLE foo
(
    bar INT NOT NULL CONSTRAINT cns DEFAULT 0
    , mar INT NOT NULL
)
GO
```

## Подсказки

💡 Pre-, post-deploy скрипты храните в отдельных папках и добавьте их в исключения линтера. Прочий код должен быть внутри хранимых процедур, триггеров и функций.
