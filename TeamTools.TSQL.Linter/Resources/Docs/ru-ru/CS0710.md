

# Арифметика с NULL возвращает значение NULL

|||
|-|-|
| Id | **CS0710**
| Мнемо | NULL_ARITHMETICS
| Серьёзность | ⚠ Предупреждение
| Категория | [СомнительноеКодирование](./Категория_СомнительноеКодирование.md)
| Исходный код | [ArithmeticsWithNullRule.cs](../../../Rules/CodeSmell/ArithmeticsWithNullRule.cs)

## Причина

Правило срабатывает, если в арифметических операциях присутствует значение NULL, что приводит к результату NULL.
<p id="descr">Проверьте значения, которые могут быть NULL.</p>

## Примеры

Некорректно

```sql
DECLARE @str VARCHAR(10)
SELECT 'abc' + @str;
```

Корректно

```sql
DECLARE @str VARCHAR(10) = 'defg'
SELECT 'abc' + @str;
```

## Подсказки

💡 Такое бывает, если забыли проинициализировать переменную или сделали копирование без учета особенностей нового фрагмента кода.

💡 Линтер предполагает, что для кода, который он анализирует, включена опция **CONCAT_WITH_NULL_YIELDS_NULL**.
