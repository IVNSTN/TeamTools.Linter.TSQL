

# Псевдоним тот же, но позиция столбца не совпадает

|||
|-|-|
| Id | **CS0723**
| Мнемо | ALIAS_MISSES_COL_POSITION
| Серьёзность | ⚠ Предупреждение
| Категория | [СомнительноеКодирование](./Категория_СомнительноеКодирование.md)
| Исходный код | [MismatchingColumnAliasPositionRule.cs](../../../Rules/CodeSmell/MismatchingColumnAliasPositionRule.cs)

## Причина

Правило срабатывает, если для обобщённого табличного выражения или подзапроса во внешнем списке столбцов присутствуют столбцы с одинаковыми именами, как в теле запроса, но расположенные в другом порядке.
<p id="descr">Проверьте имена и порядок столбцов.</p>

## Примеры

Некорректно

```sql
SELECT Id, CodeMarket
FROM
(
  SELECT TOP 10 IdMarket, CodeMarket
  FROM dbo.Markets
) AS T(CodeMarket, Id);
```

Корректно

```sql
SELECT Id, CodeMarket
FROM
(
  SELECT TOP 10 IdMarket, CodeMarket
  FROM dbo.Markets
) AS T(Id, CodeMarket);
```

## Подсказки

💡 Подзапросы и обобщённые табличные выражения допускают внешнее указание перечня возвращаемых столбцов.
