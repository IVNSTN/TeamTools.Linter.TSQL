

# Отсутствует команда GO в конце скрипта

|||
|-|-|
| Id | **CS0732**
| Мнемо | GO_AFTER_PROC
| Серьёзность | ⚠ Предупреждение
| Категория | [СомнительноеКодирование](./Категория_СомнительноеКодирование.md)
| Исходный код | [GoAfterProcRule.cs](../../../Rules/CodeSmell/GoAfterProcRule.cs)

## Причина

Правило срабатывает, если код хранимой процедуры или другого объекта в скрипте не завершается командой GO.
<p id="descr">Добавьте команду GO в конец скрипта.</p>

## Примеры

Некорректно

```sql
CREATE PROCEDURE dbo.my_proc
AS
BEGIN
...
END;
```

Корректно

```sql
CREATE PROCEDURE dbo.my_proc
AS
BEGIN
...
END;
GO
```

## Подсказки

💡 Код хранимой процедуры или другого объекта в скрипте необходимо завершать командой GO. В противном случае весь последующий код, не связанный с этим объектом, будет рассматриваться как часть его кода.

💡 Команда GO в MS SQL информирует программы SQL Server об окончании пакета инструкций Transact-SQL. Она передаёт инструментам, таким как SQL Server Management Studio (SSMS) или sqlcmd, сигнал о том, что текущий набор операторов T-SQL готов к отправке на сервер.
