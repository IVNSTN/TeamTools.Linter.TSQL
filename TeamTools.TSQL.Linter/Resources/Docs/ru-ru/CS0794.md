
# Преобразование числа в строковый литерал с поддержкой юникода

|||
|-|-|
| Id | **CS0794**
| Мнемо | NUMBER_HAS_NO_UNICODE
| Серьёзность | ℹ Подсказка
| Категория | [СомнительноеКодирование](./Категория_СомнительноеКодирование.md)
| Исходный код | [NumbersHaveNoUnicodeRule.cs](../../../Rules/CodeSmell/NumbersHaveNoUnicodeRule.cs)

## Причина

Правило срабатывает, если числовой тип явно преобразуется в `NVARCHAR` или `NCHAR`.
<p id="descr">Удалите явное преобразование в юникод.</p>

## Примеры

Некорректно

```sql
SELECT CAST(100 AS NCHAR(10));       --  1

DECLARE @i INT = -1;

PRINT CONVERT(NVARCHAR(10), @i);     -- 2
```

Корректно

```sql
DECLARE @i INT = -1;

SELECT CAST(@i AS VARCHAR(10));
```

## Подсказки

💡 Часто к юникодной строке приводят числа в длинном выражении сбора большой юникодной строки и `NVARCHAR`/`NCHAR` применены скорее для единообразия - в таких случаях исправлять ничего не нужно. В прочих же ситуациях имеет смысл проверить намерения автора, может юникод должен был возникнуть, но часть логики потерялась и осталось лишь странное преобразование числа.
