
# Символ другого алфавита в слове строкового литерала

|||
|-|-|
| Id | **CS0834**
| Мнемо | LITERAL_LOOK_ALIKE_CHAR
| Серьёзность | ℹ Подсказка
| Категория | [СомнительноеКодирование](./Категория_СомнительноеКодирование.md)
| Исходный код | [LiteralContainsLookAlikeCharRule.cs](../../../Rules/CodeSmell/LiteralContainsLookAlikeCharRule.cs)

## Причина

Правило срабатывает, если в слове строкового литерала, набранном преимущественно символами одного алфавита, обнаружен символ из другого алфавита, имеющий визуально близкий аналог в первом алфавите.
<p id="descr">Перепишите, используя символы только одного алфавита.</p>

## Примеры

Некорректно

```sql
SELECT '', NULL, 'аdsf'; -- unexpected cyrillic 'а' in latin string

DECLARE @str VARCHAR(100) = 'Cтрока'; -- unexpected latin 'C' in cyrillic string

EXEC dbo.foo @arg = 'qwerty zxсv 123'; -- unexpected cyrillic 'с' in latin string
```

Корректно

```sql
SELECT '', NULL, 'adsf'; -- latin only

DECLARE @str VARCHAR(100) = 'АБВГД'; -- cyrillic only

EXEC dbo.foo @arg = 'qwerty zxcv 123'; -- lating + numbers

PRINT 'Переменной @c присвоено неверное значение ''A''!'; -- valid mix of c/с, A/А
```

## Подсказки

💡 Строковые литералы могут использоваться не только в логике ветвления кода, в проверках на допустимые значения, но также и в качестве константных значений для вставки в таблицу, будь то вставка из хранимой процедуры или же из предеплой-скрипта. Так некорректный литерал может оказаться в данных на продакшене и оттуда его может оказаться не так-то просто устранить.
