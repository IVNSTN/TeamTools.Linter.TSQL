

# Различающиеся литералы в конструкциях INTERSECT/EXCEPT

|||
|-|-|
| Id | **CS0914**
| Мнемо | INTERSECT_EXCEPT_BROKEN_BY_LITERAL
| Серьёзность | ⚠ Предупреждение
| Категория | [СомнительноеКодирование](./Категория_СомнительноеКодирование.md)
| Исходный код | [IntersectExceptOnLiteralRule.cs](../../../Rules/CodeSmell/IntersectExceptOnLiteralRule.cs)

## Причина

Правило срабатывает, если результаты запросов, участвующих в операции INTERSECT или EXCEPT, имеют различающиеся литералы.
<p id="descr">Исправьте литералы в результатах запросов.</p>

## Примеры

Некорректно

```sql
SELECT 1 AS id, foo.a, NULL AS name FROM dbo.foo
INTERSECT
SELECT 1 AS id, bar.a, 't' AS name FROM dbo.bar
EXCEPT
SELECT 0, zar.a, 'z' FROM dbo.zar;
```

Корректно

```sql
SELECT 0 AS id, foo.a, 't' AS name FROM dbo.foo
INTERSECT
SELECT 0 AS id, bar.a, 't' AS name FROM dbo.bar
EXCEPT
SELECT 0, zar.a, 't' FROM dbo.zar;
```

## Подсказки

💡 Результаты запросов, содержащие разные литералы, не имеют смысла, или конструкция сломана.
