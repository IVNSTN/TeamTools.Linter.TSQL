

# Ожидается, что параметр является выходным (OUTPUT)

|||
|-|-|
| Id | **CS0922**
| Мнемо | PARAM_EXPECTED_AS_OUTPUT
| Серьёзность | ⚠ Предупреждение
| Категория | [СомнительноеКодирование](./Категория_СомнительноеКодирование.md)
| Исходный код | [BuiltInProcsArgumentOutputRule.cs](../../../Rules/CodeSmell/BuiltInProcsArgumentOutputRule.cs)

## Причина

Правило срабатывает, если при вызове хранимой процедуры для выходного параметра не указано предложение OUTPUT.
<p id="descr">Укажите предложение OUTPUT.</p>

## Примеры

Некорректно

```sql
CREATE PROCEDURE dbo.sp_foo
      @param1  INT = NULL
    , @param2  INT = NULL OUTPUT
...

DECLARE @p2 INT;
EXEC dbo.sp_foo @param2 = @p2;
```

Корректно

```sql
CREATE PROCEDURE dbo.sp_foo
      @param1  INT = NULL
    , @param2  INT = NULL OUTPUT
...

DECLARE @p2 INT;
EXEC dbo.sp_foo @param2 = @p2 OUTPUT;
```

## Подсказки

💡 Если явно не указать предложение OUTPUT для выходного параметра, код будет работать, но значение переменной не будет установлено.
