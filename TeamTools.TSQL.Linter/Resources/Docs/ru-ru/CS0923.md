
# Сортировка набора строк для операции вставки

|||
|-|-|
| Id | **CS0923**
| Мнемо | SORTED_INSERT
| Серьёзность | ⚠ Предупреждение
| Категория | [СомнительноеКодирование](./Категория_СомнительноеКодирование.md)
| Исходный код | [InsertWithOrderByRule.cs](../../../Rules/CodeSmell/InsertWithOrderByRule.cs)

## Причина

Правило срабатывает, если указана сортировка набора строк для вставки.
<p id="descr">Удалите сортировку набора строк.</p>

## Примеры

Некорректно

```sql
INSERT INTO dbo.table1(id, name)
SELECT id, name
FROM dbo.table2
ORDER BY id;
```

Корректно

```sql
INSERT INTO dbo.table1(id, name)
SELECT id, name
FROM dbo.table2;
```

## Подсказки

💡 Сортировка набора строк (ORDER BY) не гарантирует вставку строк в указанном порядке.

💡 Для наполнения "подготовленных" временных таблиц надежнее использовать `ROW_NUMBER`.
