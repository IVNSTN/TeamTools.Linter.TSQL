

# Переопределение входного параметра без его чтения

|||
|-|-|
| Id | **CS0924**
| Мнемо | PARAM_VALUE_IGNORED
| Серьёзность | ℹ Подсказка
| Категория | [СомнительноеКодирование](./Категория_СомнительноеКодирование.md)
| Исходный код | [ParameterValueIgnoredRule.cs](../../../Rules/CodeSmell/ParameterValueIgnoredRule.cs)

## Причина

Правило срабатывает, если значение входного параметра хранимой процедуры или функции перезаписывается без его чтения.
<p id="descr">Измените логику переопределения входного параметра.</p>

## Примеры

Некорректно

```sql
CREATE PROCEDURE dbo.sp_foo
      @param1  INT
AS
    SET @param1 = 100;
...
```

Корректно

```sql
CREATE PROCEDURE dbo.sp_foo
      @param1  INT
AS
    SET @param1 = @param1 * 100;
...
```

## Подсказки

💡 Внутри процедуры или функции должно быть ясно, почему мы изменяем входной параметр. Если переопределение параметра не имеет под собой логической основы, то этот параметр становится бесполезным.
