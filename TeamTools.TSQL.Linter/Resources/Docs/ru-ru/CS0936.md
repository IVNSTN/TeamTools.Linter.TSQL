

# Ссылка на переменную, которой ещё не присвоено значение

|||
|-|-|
| Id | **CS0936**
| Мнемо | VAR_REF_BEFORE_ASSIGNED
| Серьёзность | ⚠ Предупреждение
| Категория | [СомнительноеКодирование](./Категория_СомнительноеКодирование.md)
| Исходный код | [VariableNotAssignedBeforeUsageRule.cs](../../../Rules/CodeSmell/VariableNotAssignedBeforeUsageRule.cs)

## Причина

Правило срабатывает, если к переменной обращаются до того, как ей присвоят значение.
<p id="descr">Присвойте значение переменной.</p>

## Примеры

Некорректно

```sql
DECLARE @Id INT
SET @Id = @Id * 2
```

Корректно

```sql
DECLARE @Id INT = 5
SET @Id = @Id * 2
```

## Подсказки

💡 Ложные срабатывания возможны в циклах, где для анализа используется значение переменной из предыдущей итерации.

💡 Если в операторе DECLARE для переменной задано начальное значение NULL, это не значит, что она обязательно должна быть заполнена. Даже если ей не присваивать никакого значения явно, в ней всё равно будет храниться NULL.
