

# Последним оператором в функции должен быть RETURN

|||
|-|-|
| Id | **CS0950**
| Мнемо | FUNCTION_ENDS_WITH_RETURN
| Серьёзность | ⚠ Предупреждение
| Категория | [СомнительноеКодирование](./Категория_СомнительноеКодирование.md)
| Исходный код | [FunctionEndsWithReturnRule.cs](../../../Rules/CodeSmell/FunctionEndsWithReturnRule.cs)

## Причина

Правило срабатывает, если в конце определения функции нет оператора RETURN.
<p id="descr">Добавьте оператор RETURN.</p>

## Примеры

Некорректно

```sql
CREATE FUNCTION dbo.my_func (@id INT)
RETURNS INT
AS
BEGIN
...
END
```

Корректно

```sql
CREATE FUNCTION dbo.my_func (@id INT)
RETURNS INT
AS
BEGIN
...
RETURN (@res);
END
```

## Подсказки

💡 Это правило схоже по смыслу с правилом [CS0161](./CS0161.md), которое касается недостижимого кода(unreachable code).
