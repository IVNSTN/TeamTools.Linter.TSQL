

# Ключевое слово использовано для имени транзакции

|||
|-|-|
| Id | **CS0990**
| Мнемо | SUSPICIOUS_TRAN_NAME
| Серьёзность | ⚠ Предупреждение
| Категория | [СомнительноеКодирование](./Категория_СомнительноеКодирование.md)
| Исходный код | [SuspiciousTranNameRule.cs](../../../Rules/CodeSmell/SuspiciousTranNameRule.cs)

## Причина

Правило срабатывает, если в имени транзакции использовано ключевое слово.
<p id="descr">Присвойте транзакции имя не содержащее ключевых слов.</p>

## Примеры

Некорректно

```sql
BEGIN TRY
    BEGIN TRAN

    SELECT 1/0

    COMMIT TRAN
END TRY
BEGIN CATCH
    ROLLBACK TRAN
    THROW -- Здесь случится не проброс ошибки деления на ноль,
          -- а попытка закрыть траназакцию с именем "THROW"
END CATCH
```

Корректно

```sql
BEGIN TRY
    BEGIN TRAN

    SELECT 1/0

    COMMIT TRAN
END TRY
BEGIN CATCH
    ROLLBACK TRAN;
    THROW
END CATCH
```

## Подсказки

💡 Правило проверяет выражения:

- BEGIN TRAN
- SAVE TRAN
- COMMIT TRAN
- ROLLBACK TRAN
