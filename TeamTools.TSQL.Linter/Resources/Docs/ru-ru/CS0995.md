
# Смешивание точных типов с FLOAT или REAL

|||
|-|-|
| Id | **CS0995**
| Мнемо | FLOAT_WITH_NON_FLOAT
| Серьёзность | ℹ Подсказка
| Категория | [СомнительноеКодирование](./Категория_СомнительноеКодирование.md)
| Исходный код | [FloatMixedWithNonFloatRule.cs](../../../Rules/CodeSmell/FloatMixedWithNonFloatRule.cs)

## Причина

Правило срабатывает, если в выражении смешиваются точные типы с FLOAT или REAL.
<p id="descr">Используйте типы FLOAT или REAL с большой осторожностью.</p>

## Примеры

Некорректно

```sql
DECLARE
    @i REAL
    , @n DECIMAL(10, 2);

SELECT @i + @n;
```

Корректно

```sql
DECLARE
    @i INT
    , @n DECIMAL(10, 2);

SELECT @i + @n;
```

## Подсказки

💡 Приоритет типа `FLOAT` выше любого другого числового типа. При отсутствии явного преобразования `FLOAT`, результат любого числового выражение с участием типа `FLOAT` будет `FLOAT`.

💡 Даже если переменная типа `FLOAT` не содержит значения, её участие в формуле всё равно приводит к потере точности.
