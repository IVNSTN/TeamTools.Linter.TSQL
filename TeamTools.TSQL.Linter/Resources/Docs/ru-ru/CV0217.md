
# Порядок указания свойств столбца таблицы

|||
|-|-|
| Id | **CV0217**
| Мнемо | COLUMN_NULLABILITY_BEFORE_CONSTRAINT
| Серьёзность | ⚠ Предупреждение
| Категория | [СоглашенияПоКодированию](./Категория_СоглашенияПоКодированию.md)
| Исходный код | [ColumnNullableOptionBeforeConstraintRule.cs](../../../Rules/CodingConvention/ColumnNullableOptionBeforeConstraintRule.cs)

## Причина

Правило срабатывает, если в определении столбца таблицы не соблюден порядок следования свойств.
<p id="descr">Укажите свойства столбца в правильном порядке.</p>

## Примеры

Некорректно

```sql
CREATE TABLE foo
(
    bar    INT         IDENTITY(1, 1) NOT NULL
    , far  VARCHAR(10) CONSTRAINT ck CHECK (CONVERT(VARCHAR(3), far) = 'zar' OR far = 'jar') NOT NULL
    , mar  DATETIME    DEFAULT CONVERT(DATE, GETDATE()) NOT NULL
);
```

Корректно

```sql
CREATE TABLE foo
(
    bar    INT         NOT NULL IDENTITY(1, 1)
    , far  VARCHAR(10) NOT NULL CONSTRAINT ck CHECK (CONVERT(VARCHAR(3), far) = 'zar' OR far = 'jar')
    , mar  DATETIME    NOT NULL DEFAULT CONVERT(DATE, GETDATE())
);
```

## Подсказки

💡 Правильный порядок указания свойств столбца таблицы:

1. тип данных
2. `NULL`/`NOT NULL`
3. всё остальное (`IDENTITY`, `CONSTRAINT` и т.д.)
