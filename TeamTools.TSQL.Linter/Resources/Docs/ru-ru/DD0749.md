

# Variable-length column with small size is not recommended

|||
|-|-|

| Серьёзность | ℹ Подсказка

## Причина

<p id="descr">Колонки переменной длины помогают сэкономить место на хранении строк разного размера в столбце, которому выделено, так сказать, с запасом. Однако для управления колонками переменной длины СУБД сохраняет дополнительную информацию с указанием фактической длины каждого значения. Эта информация занимает 2 байта, и для колонки VARCHAR(2) таким образом на диске может выделяться в два раза боле места, чем если бы колонка была определена как CHAR(2), с фиксированной длиной. Если допустимо, то для колонок с короткими строками может быть более удачным выбором тип с фиксированной длиной.</p>
