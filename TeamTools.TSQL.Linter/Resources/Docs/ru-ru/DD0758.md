

# Redundant table lock escalation option

|||
|-|-|

| Серьёзность | ⚠ Предупреждение

| Исходный код | [RedundantLockEscalationSetRule.cs](../../../Rules/DatabaseDesign/RedundantLockEscalationSetRule.cs)

## Причина

<p id="descr">Реагирует на установку LOCK_ESCALATION = TABLE, поскольку TABLE это значение по умолчанию. И на установку этой опции для одной и той же таблицы несколько раз подряд. А также на установку AUTO для несекционированной таблицы, поскольку поведение в таком случае будет эквивалентно поведению при значении LOCK_ESCALATION = TABLE.</p>
