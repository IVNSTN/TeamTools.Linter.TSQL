

# Consider choosing AUTO for partitioned table

## Причина

<p id="descr">Секционированные таблицы позволяют работать с каждой секцией отдельно, что, в частности, уменьшает конкуренцию за блокировку строк и страниц. При аккуратной реализации работа всегда с отдельной партицией также снижает вероятность возникновения дедлоков. Однако при необходимости эскалации блокировок СУБД по умолчанию выберет эскалацию до уровня таблицы и выгода от разделения данных по секциям будет утрачена. Однако, в некоторых случаях, эскалация до уровня таблицы может, опять же, снизить вероятность возникновения дедлоков. Но в общем случае для секционированной таблицы стоит рассмотреть переключение в режим AUTO.</p>
