
# У таблицы отсутствует первичный ключ

|||
|-|-|
| Id | **DD0941**
| Мнемо | TABLE_HAS_PK
| Серьёзность | ℹ Подсказка
| Категория | [ДизайнБазДанных](./Категория_ДизайнБазДанных.md)
| Исходный код | [TableHasPrimaryKeyRule.cs](../../../Rules/DatabaseDesign/TableHasPrimaryKeyRule.cs)

## Причина

Правило срабатывает, если у таблицы отсутствует первичный ключ.
<p id="descr">Добавьте первичный ключ.</p>

## Примеры

Некорректно

```sql
CREATE TABLE dbo.foo
(
    id          INT NOT NULL
    , parent_id INT NULL
    , name      VARCHAR(10) NOT NULL
);
```

Корректно

```sql
CREATE TABLE dbo.foo
(
    id          INT NOT NULL PRIMARY KEY
    , parent_id INT NULL
    , name      VARCHAR(10) NOT NULL
);
```

## Подсказки

💡 Для временных таблиц и табличных переменных (# и @) тоже крайне желательно задать первичный ключ. Определение первичного ключа таблицы - такая же декларация намерения, как и определение, допускает столбец `NULL` или нет.
