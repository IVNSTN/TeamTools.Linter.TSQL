

# Возможность изменения данных с хинтом NOLOCK признана устаревшей

|||
|-|-|
| Id | **DE0819**
| Мнемо | MODIFY_WITH_NOLOCK
| Серьёзность | ⚠ Предупреждение
| Категория | [Устаревание](./Категория_Устаревание.md)
| Исходный код | [ModifyWithNolockRule.cs](../../../Rules/Deprecation/ModifyWithNolockRule.cs)

## Причина

Правило срабатывает, если в блоке FROM для DELETE или UPDATE используется хинт NOLOCK или READUNCOMMITTED.
<p id="descr">Удалите хинт NOLOCK или READUNCOMMITTED.</p>

## Примеры

Некорректно

```sql
DELETE FROM t
FROM dbo.tbl AS t (READUNCOMMITTED)
WHERE a IS NULL
```

Корректно

```sql
DELETE FROM t
FROM dbo.tbl AS t
WHERE a IS NULL
```
