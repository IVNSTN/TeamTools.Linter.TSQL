
# Количество наполняемых в FETCH переменных не совпадает с количеством столбцов в курсоре

|||
|-|-|
| Id | **FA0124**
| Мнемо | CURSOR_FETCH_COL_COUNT
| Серьёзность | ⛔ Ошибка
| Категория | [Сбой](./Категория_Сбой.md), [Курсоры](./Группа_Курсоры.md)
| Исходный код | [CursorFetchColCountRule.cs](../../../Rules/Failure/CursorFetchColCountRule.cs)

## Причина

Правило срабатывает, если количество наполняемых при чтении из курсора переменных не совпадает с количеством столбцов в курсоре.
<p id="descr">Выровняйте количество заполняемых переменных и возвращаемых курсором столбцов.</p>

## Примеры

Некорректно

```sql
DECLARE cr CURSOR FAST_FORWARD FOR
    SELECT a, b, c FROM foo;
...
FETCH NEXT FROM cr
INTO
    @a
    , @b;
...
```

Корректно

```sql
DECLARE cr CURSOR FAST_FORWARD FOR
    SELECT a, b, c FROM foo;
...
FETCH NEXT FROM cr
INTO
    @a
    , @b
    , @c;
...
```
