
# Несоответствие количества аргументов функции

|||
|-|-|
| Id | **FA0133**
| Мнемо | BUILTINS_ARG_COUNT
| Серьёзность | ⛔ Ошибка
| Категория | [Сбой](./Категория_Сбой.md)
| Исходный код | [BuiltInFunctionsArgumentCountRule.cs](../../../Rules/Failure/BuiltInFunctionsArgumentCountRule.cs)

## Причина

Правило срабатывает, если в вызываемой функции указано неверное количество аргументов.
<p id="descr">Передавайте правильное количество аргументов.</p>

## Примеры

Некорректно

```sql
DECLARE @str VARCHAR(10);

SELECT 1
WHERE 1=1 AND (1=0 OR
    LOWER() IS NULL OR
    UPPER('asdf', 'asdf') IS NULL OR
    LEFT(@str) IS NULL OR
    REPLACE(@str, 'asdf') IS NULL OR
    ISNULL(@str) IS NULL OR
    DATEADD(DAY, '1990101') IS NULL OR
    DATEADD(DAY, '1990101', 3, 3) IS NULL)
```

Корректно

```sql
DECLARE @str VARCHAR(10);

SELECT 1
WHERE 1=1 AND (1=0 OR
    LOWER('asdf') IS NULL OR
    UPPER('asdf') IS NULL OR
    LEFT(@str, 3) IS NULL OR
    REPLACE(@str, 'asdf', 'fdsa') IS NULL OR
    ISNULL(@str, 'asdf') IS NULL OR
    DATEADD(DAY, 3, '1990101') IS NULL OR
```

## Подсказки

💡 SSDT компилятор пропускает конструкции вида `ISNULL(a.name)`, которые не пропускает компилятор T-SQL.
💡 Проверяются только простые встроенные функции, у которых нет вариативности в наборе аргументов.
