
# Определение столбца производной таблицы не соответствует выводам запроса

|||
|-|-|
| Id | **FA0179**
| Мнемо | OUTPUT_COL_DEFINITION_MISMATCH
| Серьёзность | ⛔ Ошибка
| Категория | [Сбой](./Категория_Сбой.md)
| Исходный код | [ParenthesisSubqueryDefinitionColCountRule.cs](../../../Rules/Failure/ParenthesisSubqueryDefinitionColCountRule.cs)

## Причина

Правило срабатывает, если число столбцов в определении CTE не совпадает по числу столбцов с тем, что написано в самом подзапросе.
<p id="descr">Выровняйте число столбцов в определении CTE и подзапросе.</p>

## Примеры

Некорректно

```sql
WITH cte (id, dt, name, summary) AS (SELECT 1, GETDATE())
SELECT * FROM cte;
```

Корректно

```sql
WITH cte (id, name, dt) AS (SELECT 1, NULL, GETDATE())
SELECT * FROM cte;
```
