

# Целочисленный литерал находится вне диапазона

|||
|-|-|
| Id | **FA0947**
| Мнемо | LITERAL_OUT_OF_RANGE
| Серьёзность | ⛔ Ошибка
| Категория | [Сбой](./Категория_Сбой.md)
| Исходный код | [LiteralOutOfRangeRule.cs](../../../Rules/Failure/LiteralOutOfRangeRule.cs)

## Причина

Правило срабатывает, если целочисленный литерал присваивается переменной, размерность которой не позволяет его сохранить.
<p id="descr">Укажите правильную размерность переменной.</p>

## Примеры

Некорректно

```sql
DECLARE @var TINYINT;
SELECT ISNULL(@var, -1);

DECLARE @var BIT;
SELECT ISNULL(@var, -10);

DECLARE @var SMALLINT;
SELECT ISNULL(@var, 33000);
```

Корректно

```sql
DECLARE @var TINYINT;
SELECT ISNULL(@var, 0);

DECLARE @var BIT;
SELECT ISNULL(@var, 1);

DECLARE @var SMALLINT;
SELECT ISNULL(@var, 32767);
```

## Подсказки

💡 ISNULL усекает второй параметр до размера первого. Тип, размерность результата ISNULL всегда равны типу, размерности первого параметра.

💡 COALESCE, в отличие от ISNULL, вычисляет итоговые типы, размерность с учетом всех указанных аргументов.
