

# Выравнивание открывающего и закрывающего тегов многострочного комментария

|||
|-|-|
| Id | **FM0244**
| Мнемо | ML_COMMENT_OPEN_CLOSE_TAG_ALIGNED
| Серьёзность | ⚠ Предупреждение
| Категория | [Форматирование](./Категория_Форматирование.md)
| Исходный код | [MultilineCommentClosingTagAlignRule.cs](../../../Rules/Formatting/MultilineCommentClosingTagAlignRule.cs)

## Причина

Правило срабатывает, если открывающий (`/*`) и закрывающий (`*/`) теги многострочного комментария имеют разный отступ или не находятся на одной строке с текстом.
<p id="descr">Выровняйте отступ у тегов многострочного комментария.</p>

## Примеры

Некорректно

```sql
/*
Это многострочный
комментарий
    */

/* Это многострочный
комментарий
*/
```

Корректно

```sql
/*
Это многострочный
комментарий
*/

/* Это многострочный
комментарий */
```

## Подсказки

💡 Допускается два варианта оформления:

```sql
/*
"БЛОК": где открытие и закрытие оформлены
как BEGIN-END в отдельной строке и должны иметь одинаковый отступ
*/
```

или

```sql
/* "КИРПИЧ": открывающий и
закрывающий теги комментария расположены
в строках с текстом; одинаково они выровнены быть не могут */
```
