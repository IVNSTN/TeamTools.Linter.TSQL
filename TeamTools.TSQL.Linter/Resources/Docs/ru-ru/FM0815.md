

# Размерность типа данных имеет отступ от имени типа данных

|||
|-|-|
| Id | **FM0815**
| Мнемо | SPACE_BEFORE_SIZE
| Серьёзность | ⚠ Предупреждение
| Категория | [Форматирование](./Категория_Форматирование.md)
| Исходный код | [SpaceBeforeTypeSizeRule.cs](../../../Rules/Formatting/SpaceBeforeTypeSizeRule.cs)

## Причина

Правило срабатывает, если размерность типа данных имеет отступ от имени типа данных.
<p id="descr">Напишите размерность типа данных слитно с именем типа данных.</p>

## Примеры

Некорректно

```sql
DECLARE
    @d DECIMAL  (18,3)
    , @s VARCHAR/**/(MAX);
```

Корректно

```sql
DECLARE
    @d DECIMAL(18,3)
    , @s VARCHAR(MAX);
```
