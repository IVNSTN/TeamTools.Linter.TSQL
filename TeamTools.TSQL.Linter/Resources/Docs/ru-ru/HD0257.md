

# Персональные данные (email)

|||
|-|-|
| Id | **HD0257**
| Мнемо | PERSONAL_DATA_EMAIL
| Серьёзность | ⚠ Предупреждение
| Категория | [Хардкод](./Категория_Хардкод.md)
| Исходный код | [PersonalDataEmailRule.cs](../../../Rules/Hardcode/PersonalDataEmailRule.cs)

## Причина

Правило срабатывает, если в коде обнаруживается хардкод с адресом электронной почты.
<p id="descr">Удалите хардкод с адресом электронной почты.</p>

## Примеры

Некорректно

```sql
-- hello@world.org

SELECT 'adsf@asdf.ru' AS email;
```

Корректно

```sql
--@var_commented

SELECT 'adsf 926 @ asdf - .ru' AS email;
```

## Подсказки

💡Внутренние рассылки оформляйте через serv.subscription и мнемо-код. Внешние - выносите в оформление в SSSRS, движки шаблонов, хотя бы в dbo.environment

💡Похожими на почту могут оказаться закомментированная арифметика вроде *@adsf - чтобы правило не срабатывало, достаточно расставить пробелы вокруг операторов.
