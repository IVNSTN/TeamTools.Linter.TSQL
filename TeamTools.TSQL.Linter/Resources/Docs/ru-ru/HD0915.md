

# Хардкод файловой группы

|||
|-|-|
| Id | **HD0915**
| Мнемо | NON_PRIMARY_FILEGROUP
| Серьёзность | ⚠ Предупреждение
| Категория | [Хардкод](./Категория_Хардкод.md), [Индексы](./Группа_Индексы.md)
| Исходный код | [IndexFileGroupNonPrimaryRule.cs](../../../Rules/Hardcode/IndexFileGroupNonPrimaryRule.cs)

## Причина

Правило срабатывает, если в коде обнаруживается явное указание не PRIMARY файловой группы.
<p id="descr">Замените файловую группу на PRIMARY</p>

## Примеры

Некорректно

```sql
CREATE TABLE dbo.foo
(
    id INT
    , name VARCHAR(100)
    , CONSTRAINT PK_foo PRIMARY KEY CLUSTERED (id) ON [ARCHIVE]
) ON [ARCHIVE]

CREATE INDEX IX_foo ON dbo.foo(name)
ON [ARCHIVE]
```

Корректно

```sql
CREATE TABLE dbo.foo
(
    id INT
    , name VARCHAR(100)
    , CONSTRAINT PK_foo PRIMARY KEY CLUSTERED (id) ON [PRIMARY]
) ON [PRIMARY]

CREATE INDEX IX_foo ON dbo.foo(name)
ON [PRIMARY]
```

## Подсказки

💡Файловые группы - зона эксплуатации, зона ответственности ДБА. "Программировать" их не нужно.
