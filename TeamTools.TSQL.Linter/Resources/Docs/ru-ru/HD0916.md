

# Хардкод файловых групп в схеме секционирования

|||
|-|-|
| Id | **HD0916**
| Мнемо | PARTITION_SCHEME_FILEGROUPS_HARDCODED
| Серьёзность | ⚠ Предупреждение
| Категория | [Хардкод](./Категория_Хардкод.md)
| Исходный код | [PartitionSchemeFileGroupsHardcodedRule.cs](../../../Rules/Hardcode/PartitionSchemeFileGroupsHardcodedRule.cs)

## Причина

Правило срабатывает, если в определении схемы секционирования обнаруживается явное перечисление файловых групп.
<p id="descr">Замените перечисление файловых групп на ALL TO ([PRIMARY])</p>

## Примеры

Некорректно

```sql
CREATE PARTITION SCHEME [ps_ord_no]
AS PARTITION [pf_ord_no]
TO ([ARCHIVE])

CREATE PARTITION SCHEME [ps_ord_no]
AS PARTITION [pf_ord_no]
TO ([PRIMARY], [PRIMARY])
```

Корректно

```sql
CREATE PARTITION SCHEME [ps_ord_no]
AS PARTITION [pf_ord_no]
ALL TO ([PRIMARY])
```

## Подсказки

💡Файловые группы - зона эксплуатации, зона ответственности ДБА. "Программировать" их не нужно.

💡Для минимально необходимого описания схемы в проекте достаточно указать в ней

```sql
ALL TO ([PRIMARY])
```
