
# Наименование ограничения нарушает установленный шаблон

|||
|-|-|
| Id | **NM0203**
| Мнемо | CONSTRAINT_NAME_PATTERN
| Серьёзность | ⚠ Предупреждение
| Категория | [Именование](./Категория_Именование.md)
| Исходный код | [ConstraintNameFormatRule.cs](../../../Rules/Naming/ConstraintNameFormatRule.cs)

## Причина

Правило срабатывает, если наименование ограничения не соответствует установленному шаблону.
<p id="descr">Исправьте наименование ограничения согласно шаблону.</p>

## Примеры

Некорректно

```sql
ALTER TABLE pak.test ADD CONSTRAINT PK_test PRIMARY KEY (id);
```

Корректно

```sql
ALTER TABLE pak.test ADD CONSTRAINT PK_pak_test PRIMARY KEY (id);
```

## Подсказки

💡 Регистр символов учитывается

💡 Правило распространяется на ограничения уровня таблицы и столбца:

- PRIMARY KEY
- FOREIGN KEY
- DEFAULT
- UNIQUE
- CHECK
