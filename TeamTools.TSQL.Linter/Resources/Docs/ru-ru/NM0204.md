

# Написание имени переменной отличается от имени указанного при объявлении

|||
|-|-|
| Id | **NM0204**
| Мнемо | VAR_NAME_MISSPELLED
| Серьёзность | ⚠ Предупреждение
| Категория | [Именование](./Категория_Именование.md)
| Исходный код | [VariableNameSpellingRule.cs](../../../Rules/Naming/VariableNameSpellingRule.cs)

## Причина

Правило срабатывает, если написание имени переменной при обращении отличается от написания имени переменной при объявлении.
<p id="descr">Унифицируйте написание имён переменных.</p>

## Примеры

Некорректно

```sql
DECLARE @VarOne INT
SET @varone = 1;
```

Корректно

```sql
DECLARE @VarOne INT
SET @VarOne = 1;
```

## Подсказки

💡 Чтобы изменить регистр переменной (привести к стилю), можно изменить ее написание в DECLARE и применить форматтер, а не делать небезопасный REPLACE, который может задеть чуть больше чем вы хотели.

💡 Для корректного переименования переменной во всем коде без рисков можно воспользоваться функционалом Refactoring/Rename.
