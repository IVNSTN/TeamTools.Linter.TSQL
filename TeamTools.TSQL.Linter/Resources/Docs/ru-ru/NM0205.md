

# Смешение нотаций в именовании переменных

|||
|-|-|
| Id | **NM0205**
| Мнемо | VAR_NAME_NOTATION_MIX
| Серьёзность | ℹ Подсказка
| Категория | [Именование](./Категория_Именование.md)
| Исходный код | [VariableNamingNotationRule.cs](../../../Rules/Naming/VariableNamingNotationRule.cs)

## Причина

Правило срабатывает, если в коде встречаются разные стили именования переменных (PascalCase, snake_case).
<p id="descr">Приведите имена переменных к единому стилю.</p>

## Примеры

Некорректно

```sql
DECLARE @first_number INT
DECLARE @SeconNumber INT
```

Корректно

```sql
DECLARE @FirstNumber INT
DECLARE @SeconNumber INT
```

## Подсказки

💡 В случае сомнений в выборе нотации посмотрите в репозиторий, в Readme.md может присутствовать точный ответ на ваш вопрос. При невозможности изменить имена входящих параметров хранимой процедуры заведите переменные в нужном стиле, переложите в них значения параметров и по коду используйте эти переменные.
