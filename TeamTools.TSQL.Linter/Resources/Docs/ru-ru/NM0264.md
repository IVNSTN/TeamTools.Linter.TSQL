

# Нечитабельное наименование

|||
|-|-|
| Id | **NM0264**
| Мнемо | UNREADABLE_IDENTIFIER
| Серьёзность | ℹ Подсказка
| Категория | [Именование](./Категория_Именование.md)
| Исходный код | [UnreadableIdentifierRule.cs](../../../Rules/Naming/UnreadableIdentifierRule.cs)

## Причина

Правило срабатывает, если наименование объекта нечитабельно (состоит из цифр, одной буквы и т.п.).
<p id="descr">Используйте читабельные наименования объектов.</p>

## Примеры

Некорректно

```sql
DECLARE @f123 INT
CREATE TABLE [77t-@] ([x0f-9 ] INT)
```

Корректно

```sql
DECLARE @foo INT
DECLARE @i INT
```

## Подсказки

💡 В стыковочном, трансляционном, ETL коде крайне желательно отказаться от внешних имен как можно раньше. Внешняя система не предполагает соблюдение нотаций и правил именования текущей системы. Не стоит следовать особенностям реализации сторонних решений в своем проекте.

💡 Распространенные сокращения @i, @j добавлены в исключения.
