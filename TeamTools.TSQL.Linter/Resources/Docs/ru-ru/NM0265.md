

# CAPS-LOCK наименование

|||
|-|-|
| Id | **NM0265**
| Мнемо | CAPS_IDENTIFIER
| Серьёзность | ℹ Подсказка
| Категория | [Именование](./Категория_Именование.md)
| Исходный код | [CapsIdentifierRule.cs](../../../Rules/Naming/CapsIdentifierRule.cs)

## Причина

Правило срабатывает, если наименование объекта написано в верхнем регистре.
<p id="descr">Используйте для наименования закреплённую за проектом нотацию.</p>

## Примеры

Некорректно

```sql
DECLARE @VAR INT;
CREATE TABLE MY_TABLE (COL_1 INT)
```

Корректно

```sql
DECLARE @var INT;
CREATE TABLE my_table (col_1 INT)
```

## Подсказки

💡 В стыковочном, трансляционном, ETL коде крайне желательно отказаться от внешних имен как можно раньше. Внешняя система не предполагает соблюдение нотаций и правил именования текущей системы. Не стоит следовать особенностям реализации сторонних решений в своем проекте.
