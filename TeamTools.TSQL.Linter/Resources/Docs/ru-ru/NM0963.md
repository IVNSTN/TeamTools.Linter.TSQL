

# Нарушение соглашения об именовании таблиц - ожидается lower_snake_case

|||
|-|-|
| Id | **NM0963**
| Мнемо | TABLE_NAME_LOWER_SNAKE_CASE
| Серьёзность | ℹ Подсказка
| Категория | [Именование](./Категория_Именование.md)
| Исходный код | [TableViewNameLowerSnakeCaseRule.cs](../../../Rules/Naming/TableViewNameLowerSnakeCaseRule.cs)

## Причина

Правило срабатывает, если в настройках проекта указано,
что наименование таблиц должно соответствовать нотации lower_snake_case,
но оно не соответствует.

<p id="descr">Используйте нотацию lower_snake_case.</p>

## Примеры

Некорректно

```sql
CREATE TABLE dbo.regCustomers (id BIGINT, name NVARCHAR(50));
```

Корректно

```sql
CREATE TABLE dbo.reg_customers (id BIGINT, name NVARCHAR(50));
```

## Подсказки

💡 Правило также распространяется на временные таблицы # и табличные переменные @

💡 Требует такую нотацию только в проектах, где это явно указано в настройках.
