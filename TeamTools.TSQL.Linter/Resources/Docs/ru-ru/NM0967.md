

# Нарушение соглашения об именовании полей - ожидается PascalCase

|||
|-|-|
| Id | **NM0967**
| Мнемо | FIELD_NAME_PASCAL_CASE
| Серьёзность | ℹ Подсказка
| Категория | [Именование](./Категория_Именование.md)
| Исходный код | [FieldNamePascalCaseRule.cs](../../../Rules/Naming/FieldNamePascalCaseRule.cs)

## Причина
Правило срабатывает, если в настройках проекта указано, 
что наименование колонок должно соответствовать нотации PascalCase,
но оно не соответствует.

<p id="descr">Используйте нотацию PascalCase.</p>

## Примеры

Некорректно

```sql
CREATE TABLE dbo.reg_customers (id BIGINT, name NVARCHAR(50), city_id INT);
```

Корректно

```sql
CREATE TABLE dbo.reg_customers (id BIGINT, name NVARCHAR(50), CityId INT);
```

## Подсказки

💡 Правило также распространяется на вью, табличные функции, процедуры, временные таблицы # и табличные переменные @

💡 Требует такую нотацию только в проектах, где это явно указано в настройках.
