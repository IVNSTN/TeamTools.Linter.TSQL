

# Нарушение соглашения об именовании переменных - ожидается camelCase

|||
|-|-|
| Id | **NM0972**
| Мнемо | VARIABLE_NAME_CAMEL_CASE
| Серьёзность | ℹ Подсказка
| Категория | [Именование](./Категория_Именование.md)
| Исходный код | [VariableNameCamelCaseRule.cs](../../../Rules/Naming/VariableNameCamelCaseRule.cs)

## Причина

Правило срабатывает, если в настройках проекта указано,
что наименование переменных должно соответствовать нотации camelCase,
но оно не соответствует.

<p id="descr">Используйте нотацию camelCase.</p>

## Примеры

Некорректно

```sql
DECLARE @my_var NVARCHAR(50);
```

Корректно

```sql
DECLARE @myVar NVARCHAR(50);
```

## Подсказки

💡 Табличные переменные подчиняются правилам для таблиц.

💡 Требует такую нотацию только в проектах, где это явно указано в настройках.
