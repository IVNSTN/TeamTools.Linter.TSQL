

# DDL вперемешку с DML

|||
|-|-|
| Id | **PF0875**
| Мнемо | DDL_DML_MIX
| Серьёзность | ℹ Подсказка
| Категория | [Производительность](./Категория_Производительность.md)
| Исходный код | [MixOfDdlAndDmlRule.cs](../../../Rules/Performance/MixOfDdlAndDmlRule.cs)

## Причина

Правило срабатывает, если DDL и DML команды в запросе перемешаны.
<p id="descr">Переместите все DDL команды в начало запроса.</p>

## Примеры

Некорректно

```sql
CREATE PROC dbo.delete_data
AS
BEGIN
    CREATE TABLE #t (id INT);

    SELECT br.id FROM dbo.bar AS br INNER JOIN #t AS tt ON tt.id = br.parent_id;

    CREATE INDEX ix ON #t (id);

    DELETE bar FROM dbo.bar AS br INNER JOIN #t AS tt ON tt.id = br.parent_id;
END;
GO
```

Корректно

```sql
CREATE PROC dbo.delete_data
AS
BEGIN
    CREATE TABLE #t (id INT);

    CREATE INDEX ix ON #t (id);

    SELECT br.id FROM dbo.bar AS br INNER JOIN #t AS tt ON tt.id = br.parent_id;

    DELETE bar FROM dbo.bar AS br INNER JOIN #t AS tt ON tt.id = br.parent_id;
END;
GO
```

## Подсказки

💡 Правило игнорирует запросы без источника такие как STRING_SPLIT, OPENJSON, OPENXML и манипуляции над табличными переменными.
