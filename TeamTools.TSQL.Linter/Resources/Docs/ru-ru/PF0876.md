

# Вместо фильтра по диапазону применена функция преобразования даты

|||
|-|-|
| Id | **PF0876**
| Мнемо | EXPAND_DATE_FUNC
| Серьёзность | ⚠ Предупреждение
| Категория | [Производительность](./Категория_Производительность.md)
| Исходный код | [ExpandDateFunctionRule.cs](../../../Rules/Performance/ExpandDateFunctionRule.cs)

## Причина

Правило срабатывает, если в предикате вместо фильтра по диапазону применена функция преобразования даты.
<p id="descr">Используйте предикат по диапазону вместо преобразования даты.</p>

## Примеры

Некорректно

```sql
SELECT sum(payment)
FROM taxes
WHERE YEAR(pay_period) = @tax_year;
```

Корректно

```sql
SELECT sum(payment)
FROM taxes
WHERE pay_period >= @tax_year_begin
    AND pay_period < @next_tax_year
```
