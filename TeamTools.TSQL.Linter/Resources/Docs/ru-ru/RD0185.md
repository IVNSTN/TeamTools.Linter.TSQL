

# Избыточное указание ограничения DEFAULT NULL

|||
|-|-|
| Id | **RD0185**
| Мнемо | DEFAULT_NULL
| Серьёзность | ⚠ Предупреждение
| Категория | [Избыточность](./Категория_Избыточность.md)
| Исходный код | [NullDefaultNullRule.cs](../../../Rules/Redundancy/NullDefaultNullRule.cs)

## Причина

<p id="descr">Для NOT NULL колонок в качестве дефолта выбрать NULL - ошибка, для NULLable - излишне. NULL сам себя вполне успешно вставит.</p>

## Примеры

Некорректно

```sql
CREATE TABLE dbo.employee(id INT NOT NULL, salary INT NULL DEFAULT NULL)
```

Корректно

```sql
CREATE TABLE dbo.employee(id INT NOT NULL, salary INT NULL)
```

## Подсказки

💡 Остается шанс что где-то при заполнении этого поля используется ключевое слово DEFAULT, тогда при сносе дефолта такой код сломается. Но это странный сценарий. NULL можно и явно вставить, из инсерта и вовсе можно поле убрать - NULL сам получится. Ради такого кода дефолт сохранять не нужно, лучше откорректировать сам, опирающийся на ненужный дефолт, код.
