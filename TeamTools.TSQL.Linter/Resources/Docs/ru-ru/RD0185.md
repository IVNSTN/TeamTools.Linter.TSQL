
# Избыточное указание ограничения DEFAULT NULL

|||
|-|-|
| Id | **RD0185**
| Мнемо | DEFAULT_NULL
| Серьёзность | ⚠ Предупреждение
| Категория | [Избыточность](./Категория_Избыточность.md)
| Исходный код | [NullDefaultNullRule.cs](../../../Rules/Redundancy/NullDefaultNullRule.cs)

## Причина

Правило срабатывает, если для `NOT NULL` либо для NULLable столбца установлено значение по умолчанию `NULL`.
<p id="descr">Удалите значение по умолчанию 'NULL'.</p>

## Примеры

Некорректно

```sql
CREATE TABLE dbo.employee(id INT NOT NULL, salary INT NULL DEFAULT NULL)
```

Корректно

```sql
CREATE TABLE dbo.employee(id INT NOT NULL, salary INT NULL)
```

## Подсказки

💡 Остается шанс что где-то при заполнении этого столбца используется ключевое слово `DEFAULT`, тогда при удалении ограничения такой код сломается. Но это странный сценарий. `NULL` можно и явно вставить, или вовсе убрать столбец из `INSERT` - `NULL` сам получится. Ради такого кода ограничение сохранять не нужно, лучше откорректировать этот, опирающийся на ненужный `DEFAULT`, код.
