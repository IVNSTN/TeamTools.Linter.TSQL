
# Бессмысленная попытка контролировать вывод EXISTS

|||
|-|-|
| Id | **RD0192**
| Мнемо | EXISTS_OUTPUT_CONTROL
| Серьёзность | ⚠ Предупреждение
| Категория | [Избыточность](./Категория_Избыточность.md)
| Исходный код | [ExistsOutputControlRule.cs](../../../Rules/Redundancy/ExistsOutputControlRule.cs)

## Причина

Правило срабатывает, если в подзапросе `EXISTS` используется возврат списка столбцов, предложения `TOP`, `ORDER BY`, `GROUP BY` без `HAVING` или `OFFSET`.
<p id="descr">Замените список столбцов на SELECT 1, удалите лишние инструкции.</p>

## Примеры

Некорректно

```sql
IF EXISTS (SELECT TOP(10) id FROM dbo.employee ORDER BY salary)...
```

Корректно

```sql
IF EXISTS (SELECT 1 FROM dbo.employee)...
```

## Подсказки

💡 При обработке `EXISTS` DB Engine полностью игнорирует часть `SELECT`, поэтому не нужно ни столбцов, ни `TOP`. Списка он тоже вернуть не сможет, поэтому не нужны и `ORDER BY`, `GROUP BY` (без `HAVING`), `OFFSET`.
