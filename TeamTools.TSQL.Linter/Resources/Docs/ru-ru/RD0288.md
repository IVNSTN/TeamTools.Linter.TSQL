

# Избыточный выбор скаляра

|||
|-|-|
| Id | **RD0288**
| Мнемо | REDUNDANT_SELECT_SCALAR
| Серьёзность | ⚠ Предупреждение
| Категория | [Избыточность](./Категория_Избыточность.md)
| Исходный код | [RedundantSelectScalarRule.cs](../../../Rules/Redundancy/RedundantSelectScalarRule.cs)

## Причина

<p id="descr">Заворачивать скалярные переменные и литералы в (SELECT ...) совершенно незачем.</p>

## Примеры

Некорректно

```sql
INSERT dbo.foo (title)
SELECT @title
WHERE @title IS NOT NULL
```

Корректно

```sql
IF @title IS NOT NULL
BEGIN
    INSERT dbo.foo(title)
    VALUES (@title)
END
```

## Подсказки

💡 Про ORDER BY (SELECT NULL) правило в курсе и на него не ругается. Хоть это и не самая желанная конструкция в T-SQL.

💡 На CROSS/OUTER APPLY в которые для соблюдения DRY выносятся формулы правило тоже не реагирует.
