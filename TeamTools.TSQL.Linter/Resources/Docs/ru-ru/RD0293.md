

# Избыточное указание ELSE NULL

|||
|-|-|
| Id | **RD0293**
| Мнемо | REDUNDANT_CASE_ELSE_NULL
| Серьёзность | ⚠ Предупреждение
| Категория | [Избыточность](./Категория_Избыточность.md)
| Исходный код | [RedundantCaseElseNullRule.cs](../../../Rules/Redundancy/RedundantCaseElseNullRule.cs)

## Причина

<p id="descr">В блоке CASE явно прописывать опцию ELSE NULL не нужно - NULL и так вернется, если ни один из вариантов не подошел.</p>

## Примеры

Некорректно

```sql
SELECT CASE WHEN @a = 5 THEN 4 ELSE NULL END;
```

Корректно

```sql
SELECT CASE WHEN @a = 5 THEN 4 END;
```

## Подсказки

💡 Существует и противоположная точка зрения о том, что все варианты нужно явно декларировать. Обе имеют одинаково шаткую позицию, выбор сделан в пользу одного конкретного.
