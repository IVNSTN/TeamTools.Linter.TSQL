

# Избыточное преобразование типов

|||
|-|-|
| Id | **RD0706**
| Мнемо | REDUNDANT_TYPE_CONVERSION
| Серьёзность | ⚠ Предупреждение
| Категория | [Избыточность](./Категория_Избыточность.md)
| Исходный код | [RedundantTypeConversionRule.cs](../../../Rules/Redundancy/RedundantTypeConversionRule.cs)

## Причина

<p id="descr">Если переменная или выражение уже имеют, к примеру, тип INT или VARCHAR(10), то явно преобразовывать их к этому же типу незачем.</p>

## Примеры

Некорректно

```sql
DECLARE @b VARCHAR(1) = 'B'

PRINT CAST('A' + @b AS VARCHAR(2))
```

Корректно

```sql
DECLARE @b VARCHAR(1) = 'B'

PRINT 'A' + @b
```

## Подсказки

💡 Литералы правило игнорирует, поскольку явное указание, какой тип должна иметь, к примеру, вычисляемая колонка в запросе, это скорее хороший стиль.
