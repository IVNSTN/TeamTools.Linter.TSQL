

# Использование CHECK CONSTRAINT вместо атрибута колонки

|||
|-|-|
| Id | **RD0927**
| Мнемо | REDUNDANT_COL_NULLABILITY_CHECK
| Серьёзность | ⚠ Предупреждение
| Категория | [Избыточность](./Категория_Избыточность.md)
| Исходный код | [RedundantCheckColNullabilityRule.cs](../../../Rules/Redundancy/RedundantCheckColNullabilityRule.cs)

## Причина

Правило срабатывает, если обнаружено определение CHECK CONSTRAINT для проверки на NULL/NOT NULL.
<p id="descr">Удалите CHECK CONSTRAINT, используйте атрибут колонки.</p>

## Примеры

Некорректно

```sql
CREATE TABLE dbo.foo
(
    id          INT NOT NULL CHECK (id IS NOT NULL),
    parent_id   INT CHECK (parent_id IS NULL),
    title       VARCHAR(100),
    CONSTRAINT ck_title CHECK(title IS NOT NULL)
)
```

Корректно

```sql
CREATE TABLE dbo.foo
(
    id          INT NOT NULL,
    parent_id   INT NULL,
    title       VARCHAR(100) NOT NULL
)
```
