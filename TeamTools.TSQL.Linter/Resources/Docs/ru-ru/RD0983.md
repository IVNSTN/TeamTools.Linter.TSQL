
# Извлекаемое из курсора значение не используется

|||
|-|-|
| Id | **RD0983**
| Мнемо | CURSOR_FETCHED_VAR_UNUSED
| Серьёзность | ⚠ Предупреждение
| Категория | [Избыточность](./Категория_Избыточность.md)
| Исходный код | [CursorFetchedVarUnusedRule.cs](../../../Rules/Redundancy/CursorFetchedVarUnusedRule.cs)

## Причина

Правило срабатывает, если обнаружено неиспользуемое значение, извлекаемое из курсора.
<p id="descr">Удалите неиспользуемое значение, извлекаемое из курсора.</p>

## Примеры

Некорректно

```sql
DECLARE cr CURSOR FAST_FORWARD FOR
    SELECT id, title FROM dbo.foo;
OPEN cr;

FETCH NEXT FROM cr INTO @id, @title;

PRINT @title;

CLOSE cr;
DEALLOCATE cr;
```

Корректно

```sql
DECLARE cr CURSOR FAST_FORWARD FOR
    SELECT id, title FROM dbo.foo;
OPEN cr;

FETCH NEXT FROM cr INTO @id, @title;

SET @id += 1;
PRINT @title;

CLOSE cr;
DEALLOCATE cr;
```

## Подсказки

💡 Вместе с удалением неиспользуемого значения, возможно, станет не нужна и переменная, и, вероятно, удастся упростить и запрос, который больше не должен возвращать фактически неиспользуемые столбцы.
