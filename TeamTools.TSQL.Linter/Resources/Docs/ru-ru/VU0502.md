

# Неожидаемое изменение настроек базы данных

|||
|-|-|
| Id | **VU0502**
| Мнемо | DATABASE_OPTION_SWITCH
| Серьёзность | ⚠ Предупреждение
| Категория | [Уязвимость](./Категория_Уязвимость.md)
| Исходный код | [DatabaseOptionSwitchRule.cs](../../../Rules/Vulnerability/DatabaseOptionSwitchRule.cs)

## Причина

Правило срабатывает при обнаружении команд, меняющих настройки базы данных.
<p id="descr">Не используйте код, меняющий настройки БД.</p>

## Примеры

Некорректно

```sql
ALTER DATABASE mydb SET RECOVERY FULL;

SET TEXTSIZE 12;

EXEC sp_configure 'show advanced options', '1';
```

## Подсказки

💡 Для служебных и административных процедур выбирайте специально отведенные для этого схемы и БД.

💡 Получив доступ к служебной процедуре, рядовой пользователь продукта может прямо или косвенно получить доступ к управлению СУБД, что небезопасно. Только администраторы БД могут иметь доступ такого уровня.
