

# Небезопасное изменение привелегий пользователя

|||
|-|-|
| Id | **VU0512**
| Мнемо | PRIVILEGE_MANAGEMENT
| Серьёзность | ⚠ Предупреждение
| Категория | [Уязвимость](./Категория_Уязвимость.md)
| Исходный код | [PrivilegeManagementInCodeRule.cs](../../../Rules/Vulnerability/PrivilegeManagementInCodeRule.cs)

## Причина

Правило срабатывает при обнаружении команд для изменения привелегий пользователям.
<p id="descr">Удалите команды управления доступом.</p>

## Примеры

Некорректно

```sql
CREATE ROLE test_role;

ALTER ROLE test_role ADD MEMBER usr;

GRANT EXEC on dbo.test TO usr;
```

## Подсказки

💡 Для служебных и административных процедур выбирайте специально отведенные для этого схемы и БД.

💡 Получив доступ к служебной процедуре, рядовой пользователь продукта может прямо или косвенно получить доступ к управлению СУБД, что небезопасно. Только администраторы БД могут иметь доступ такого уровня.

💡 Для контроля доступа могут быть использованы специальные таблицы, предназначенные для описания разрешений и ограничений к объектам системы (а не объектам базы данных).
