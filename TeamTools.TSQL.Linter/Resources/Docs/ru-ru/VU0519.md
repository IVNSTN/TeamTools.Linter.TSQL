

# Запрещенное использование системных динамических представлений

|||
|-|-|
| Id | **VU0519**
| Мнемо | SYS_DYNAMIC_VIEW
| Серьёзность | ⚠ Предупреждение
| Категория | [Уязвимость](./Категория_Уязвимость.md)
| Исходный код | [DynamicViewsRule.cs](../../../Rules/Vulnerability/DynamicViewsRule.cs)

## Причина

Правило срабатывает при обнаружении системных динамических представлений.
<p id="descr">Перепишите код без использования системных динамических представлений.</p>

## Примеры

Некорректно

```sql
SELECT * FROM sys.dm_exec_query_stats;
```

## Подсказки

💡 Для служебных и административных процедур выбирайте специально отведенные для этого схемы и БД.

💡 Получив доступ к служебной процедуре, рядовой пользователь продукта может прямо или косвенно получить доступ к управлению СУБД, что небезопасно. Только администраторы БД могут иметь доступ такого уровня.

💡 В большинстве случаев системные динамические представления требуют повышенных привилегий.

💡 Для целей логирования должно быть достаточно функции HOST_NAME(), IP адрес извлекать не нужно.
