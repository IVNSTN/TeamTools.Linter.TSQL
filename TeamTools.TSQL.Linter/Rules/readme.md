
# Особенности написания правил

Правила должны быть **immutable**! Т.е. анализ любого скрипта не должен оставлять в правиле никаких следов, не должен иметь влияния на анализ каких-то других скриптов.
Таким образом правило может хранить только общие данные, относящиеся к анализу любых скриптов. Например, перечень названий ф-й, на которые оно реагирует.

Нужно в том числе для того чтобы инстанцирвать правила один раз и выполнить на множестве файлов. В том числе в параллельных тредах.
